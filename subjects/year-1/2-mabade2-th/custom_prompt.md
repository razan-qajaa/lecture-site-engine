## القواعد الإلزامية

- لا تتجاهل أي سطر أو معلومة وردت في المحاضرة
- أكمل الناقص مع وسم **(شرح زيادة للفهم)** أو **(غير مشروحة في المحاضرة)**
- ابدأ من المبتدئ، لا تنتقل لنقطة قبل إتمام شرح السابقة
- اشرح **لماذا** وراء كل فكرة، لا التعريف فقط
- تشبيه يومي + مثال عملي بعد كل نقطة
- اتبع تسلسل المحاضرة نفسها
- حلّل نوع المحاضرة قبل البدء وطبّق القالب المناسب

- إذا كانت عن **الجدولة (Scheduling)** → استخدم `جداول حسابية`، `معادلات`، `رسوم بيانية`

## 🎯 أنواع أسئلة إضافية إلزامية للأتمتة (الفحص بالكامل آلي)

> هذه الأنواع تُدمَج **ضمن** الأجزاء أعلاه (لا تُنشئ لها أقساماً ## منفصلة) — التزم بها عند توليد كل جزء لضمان كفاية وتنوّع الأسئلة القابلة للتصحيح الآلي.

### مراجعة سريعة (سؤال / جواب) — ضمن qa_cards / qa_inline / theory (≥14، easy)
**Q{N}:** سؤال مراجعة مباشر (تعريف، فرق، متى نستخدم)
**A:** إجابة مختصرة قابلة للمقارنة الآلية (جملة أو جملتان)
— بدون تعليل طويل. مناسب للحفظ السريع قبل الامتحان.
مثال: «ما الفرق بين `RAM` و `ROM`؟» / «متى نستخدم `FCFS` بدل `RR`؟»

### تحدي مركّب (سيناريو + أسئلة متعددة) — ضمن mcq / debug / theory (≥3، hard)
### السيناريو {N}: {عنوان}
> [وصف الحالة: كود، جدول، مخطط، أو مسألة رياضية كاملة — 5–10 أسطر]
ثم 3–5 أسئلة MCQ **كلها مبنية على نفس السيناريو** (لا تكرر نفس الفكرة).
المصادر: `code_trace`، `numerical_case`، `algorithm_state`، `logic_entailment`، `system_design`.
مثال: «جدول تكرارات خوارزمية جدولة ناقص — أي قيمة في الخلية X؟» / «كود معالجة مقاطعة — أي عقدة تُقصى؟»

### تطبيق عملي (أكمل / صحّح / نفّذ) — ضمن exercise / analysis_exercise / debug (≥5، medium)
كل تمرين: **المعطيات** + **المطلوب** (مرقّم) + **نموذج الحل الكامل**
الطالب يجيب بقيمة/جدول/كود — الإجابة قابلة للمقارنة حرفياً أو رقمياً.
الأنواع: `fill_gaps`، `code_fix`، `table_fill`، `numerical_solve`، `diagram_completion`.

### تتبع خطوة بخطوة (جدول ناقص) — ضمن trace_exercise / mcq (≥4، medium)
**المدخل:** {حالة ابتدائية}
**أكمل الجدول:**
| الخطوة | العملية | الحالة |
| 1 | ... | ؟ |
**نموذج الحل:** جدول مكتمل + **النتيجة النهائية**
+ 1–2 سؤال MCQ على نفس التتبع (ماذا في الخطوة 3؟)
الأهداف: `algorithm_iterations`، `variable_state`، `search_tree_expansion`، `instruction_cycle_state`.

### مقارنة وتحليل (أيهما أفضل؟ لماذا؟) — ضمن mcq / theory / analysis_exercise (≥4، medium)
سؤال مقارنة بخيارات (أ، ب، ج، د) + تعليل لماذا الباقي خاطئ
أو جدول: | المعيار | الخيار A | الخيار B |
أو تمرين تحليل: «في الحالة X استخدم Y لأن Z»

### اكتشف الخطأ (كود / منطق / حساب) — ضمن debug / exercise (≥4، hard)
**الكود/الحساب (يحتوي خطأ):** …
**اكتشف الخطأ:** [وصف موجز]
**التصحيح:** … + **شرح الحل** (3 نقاط)
أنواع الأخطاء: `logic`، `misconception`، `off_by_one`، `wrong_formula`، `race_condition`، `infinite_loop`.

### نظري منظم (تعريف + مكونات + مثال) — ضمن theory / mcq (≥10، medium)
### السؤال {N}: {السؤال}
**نموذج الإجابة:**
1. التعريف: …
2. المكونات/الشروط: …
3. مثال: …
4. متى نستخدم: …
— هيكل ثابت يسهّل التصحيح الآلي بالكلمات المفتاحية.

## قواعد الكتل داخل الشرح

**📊 المخطط** — 3 أقسام بالترتيب: ما هذا المخطط؟ + جدول العُقد + جدول الروابط + بلوك `diagram`.
types: `flowchart`، `decision-tree`، `dfd`، `usecase`، `class`، `activity`. انظر SCHEMA.md v1.0 §Diagram.

**⚙️ الخطوات / الخوارزمية** — أسطر داخل fence ` ```algorithm ` بصيغة `1 | الخطوة | الأداة | ماذا يحدث` (سطر لكل خطوة). لا ترسم مربعات ولا أسهم — البارسر يحوّلها. انظر SCHEMA.md v1.0 §Algorithm.

pseudo-code في code fence بـ `language_tag=text`.

**الفهم الخاطئ ❌** / **الفهم الصحيح ✅** — سطر واحد لكل منهما.

#### مهم للامتحان ⚠️ / #### نقطة مهمة ⚠️ / #### ملاحظة: / #### الدرس المستفاد:
← `blockquote` يتبع كل منها مباشرةً.

**🤔 تفعيل الفهم** — استخدمه **≥3 مرات** في كل محاضرة.

كود ناقص — ضع `_______` أو `// (N)` مكان السطر الناقص.

تصحيح كود — قدّم الكود الخاطئ أولاً ثم المصحّح.

**💡 التشبيه** — بعد كل مفهوم مجرّد: جملة واحدة من الحياة اليومية + «وجه الشبه: X = Y». استخدمه بكثرة. انظر SCHEMA.md v1.0 §Analogy.

**⚖️ المقايضة** — جدول: المزايا | العيوب | متى تختاره. للحالات التي لا يوجد فيها خطأ وصواب — لا تستخدم compare block هنا. انظر SCHEMA.md v1.0 §Trade-off.

**🔄 قبل / بعد** — كود/حالة قبل العملية + بعدها + جملة «ماذا تغيّر؟». انظر SCHEMA.md v1.0 §Before-After.

**🔍 تتبع التنفيذ** (inline) — المدخل + جدول الخطوات (أعمدته تختلف حسب الموضوع) + النتيجة. انظر SCHEMA.md v1.0 §Trace.

**📐 المعادلة** — LaTeX داخل `$$` أو fence `math`. يتبعها **الشرح:** بمعنى كل رمز (`Average Waiting Time`، `Average Turnaround Time`، `Response Time`، `Throughput`، `CPU Utilization`). استخدم `$...$` للصيغ القصيرة داخل الفقرات. انظر SCHEMA.md v1.0 §Equation.

```
### قوالب الكتل

#### ◈ block-diagram (diagram & uml)

```
#### 📊 المخطط: {title}

#### ما هذا المخطط؟
> [ماذا يوضّح؟ لماذا وُجد؟]

#### وصف العُقد:
| # | العُقدة | النوع `kind` | الشرح |
| --- | --- | --- | --- |

#### وصف الروابط:
| من | إلى | التسمية | نوع السهم | الشرح |
| --- | --- | --- | --- | --- |

```diagram
type: {diagram_type}
title: {title}
direction: TD
nodes:
  - id: start
    label: Start
    kind: event
    level: 0
edges:
  - from: start
    to: next
```
