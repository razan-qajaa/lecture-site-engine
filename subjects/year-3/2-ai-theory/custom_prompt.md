## القواعد الإلزامية

- لا تتجاهل أي سطر أو معلومة وردت في المحاضرة
- أكمل الناقص مع وسم **(شرح زيادة للفهم)** أو **(غير مشروحة في المحاضرة)**
- ابدأ من المبتدئ، لا تنتقل لنقطة قبل إتمام شرح السابقة
- اشرح **لماذا** وراء كل فكرة، لا التعريف فقط
- تشبيه يومي + مثال عملي بعد كل نقطة
- اتبع تسلسل المحاضرة نفسها
- حلّل نوع المحاضرة قبل البدء وطبّق القالب المناسب

## التكيف مع نوع المحاضرة

قبل الشرح، حدّد فئة المحاضرة وطبّق القالب المناسب في الجزء الأول:

- إذا كانت عن **Introduction** → استخدم `Turing Test`، `Cognitive Science`، `Laws of Thought`، `Rational Agent`
- إذا كانت عن **Intelligent Agents** → استخدم `PEAS`، `Environment Types`، `Agent Function`، `Rationality`
- إذا كانت عن **Problem Solving and Search** → استخدم `BFS`، `DFS`، `Uniform Cost`، `IDS`، `A*`، `Greedy`
- إذا كانت عن **Local Search Algorithms** → استخدم `Hill Climbing`، `Simulated Annealing`، `Genetic Algorithms`
- إذا كانت عن **Game Playing** → استخدم `Minimax`، `Alpha-Beta Pruning`، `Utility`
- إذا كانت عن **Logical Agent** → استخدم `Propositional Logic`، `Entailment`، `Resolution`، `Forward Chaining`، `Backward Chaining`
- إذا كانت عن **First-Order Logic** → استخدم `Syntax`، `Semantics`، `Quantifiers`، `Unification`
- إذا كانت عن **Inference in First-Order Logic** → استخدم `Universal Instantiation`، `Existential Instantiation`، `Forward Chaining`، `Backward Chaining`، `Resolution`
- إذا كانت عن **Machine Learning** → استخدم `Supervised`، `Unsupervised`، `Reinforcement`، `Representation`، `Optimization`، `Evaluation`
- إذا كانت عن **Linear Regression** → استخدم `Cost Function`، `Gradient Descent`، `Learning Rate`، `Convergence`
- إذا كانت عن **Rule Learning** → استخدم `Divide and Conquer`، `Rule Induction`، `Pruning`

## 🎯 أنواع أسئلة إضافية إلزامية للأتمتة (الفحص بالكامل آلي)

> هذه الأنواع تُدمَج **ضمن** الأجزاء أعلاه (لا تُنشئ لها أقساماً ## منفصلة) — التزم بها عند توليد كل جزء لضمان كفاية وتنوّع الأسئلة القابلة للتصحيح الآلي.

### مراجعة سريعة (سؤال / جواب) — ضمن qa_cards / qa_inline / theory (≥16، easy)
**Q{N}:** سؤال مراجعة مباشر (تعريف، فرق، متى نستخدم)
**A:** إجابة مختصرة قابلة للمقارنة الآلية (جملة أو جملتان)
— بدون تعليل طويل. مناسب للحفظ السريع قبل الامتحان.
مثال: «ما الفرق بين `BFS` و `DFS`؟» / «متى نفضّل `A*` على `Greedy Best-First`؟»

### تحدي مركّب (سيناريو + أسئلة متعددة) — ضمن mcq / debug / theory (≥3، hard)
### السيناريو {N}: {عنوان}
> [وصف الحالة: شجرة بحث، pseudocode، جدول حالات، أو مسألة منطقية كاملة — 5–10 أسطر]
ثم 3–5 أسئلة MCQ **كلها مبنية على نفس السيناريو** (لا تكرر نفس الفكرة):
### السؤال {N}.1 (hard) … أ) ب) ج) د) + تعليل
كل سيناريو يختبر: فهم + تطبيق + قرار (أي عقدة تالية؟ أي ناتج؟ أين الخطأ؟)
المصادر: `code_trace`، `algorithm_state`، `logic_entailment`، `system_design`.
مثال: «شجرة بحث `A*` ناقصة — أي عقدة تُوسَّع تالياً؟» / «كود `Minimax` + alpha-beta — أي عقدة تُقصى؟» / «قواعد منطقية — أي استنتاج صحيح؟»

### تطبيق عملي (أكمل / صحّح) — ضمن exercise / analysis_exercise / debug (≥5، medium)
كل تمرين: **المعطيات** + **المطلوب** (مرقّم) + **نموذج الحل الكامل**
الطالب يجيب بقيمة/جدول/pseudocode — الإجابة قابلة للمقارنة حرفياً أو رقمياً.
الأنواع: `fill_gaps`، `code_fix`، `search_trace`، `logic_table`، `minimax_trace`، `heuristic_eval`.

### تتبع خطوة بخطوة — ضمن trace_exercise / mcq (≥4، medium)
**المدخل:** {حالة ابتدائية / شجرة بحث / قاعدة معرفة}
**أكمل الجدول:**
| الخطوة | العملية | الحالة |
| 1 | ... | ؟ |
**نموذج الحل:** جدول مكتمل + **النتيجة النهائية**
+ 1–2 سؤال MCQ على نفس التتبع (ماذا في الخطوة 3؟)
الأهداف: `algorithm_iterations`، `search_tree_expansion`، `variable_state`.

### مقارنة وتحليل — ضمن mcq / theory / analysis_exercise (≥4، medium)
سؤال مقارنة بخيارات (أ، ب، ج، د) + تعليل لماذا الباقي خاطئ
أو جدول: | المعيار | الخيار A | الخيار B |
أو تمرين تحليل: «في الحالة X استخدم الخوارزمية Y لأن Z»

### اكتشف الخطأ (pseudocode / منطق) — ضمن debug / exercise (≥4، hard)
**الكود/الحساب (يحتوي خطأ):** …
**اكتشف الخطأ:** [وصف موجز]
**التصحيح:** … + **شرح الحل** (3 نقاط)
أنواع الأخطاء: `logic`، `misconception`، `infinite_loop`، `wrong_heuristic`.

### نظري منظم — ضمن theory / mcq (≥10، medium)
### السؤال {N}: {السؤال}
**نموذج الإجابة:**
1. التعريف: …
2. المكونات/الشروط: …
3. مثال: …
4. متى نستخدم: …
— هيكل ثابت يسهّل التصحيح الآلي بالكلمات المفتاحية.

## قواعد الكتل داخل الشرح

**💻 الكود:** `pseudocode` — داخل كل كود: تعليق إنجليزي لكل سطر. يتبعه **شرح كل سطر**. انظر SCHEMA.md v1.0 §Code.

**شرح كل سطر:** format=`numbered_list`. انظر SCHEMA.md v1.0 §Line-explain.

**الناتج المتوقع** — `blockquote` بعد كل كود (ترتيب العقد، الحل، القيمة، جدول البحث).

**🛠️ استكشاف الأخطاء** — جدول: الخطأ | السبب | الحل.

**📊 المخطط** — 3 أقسام بالترتيب: ما هذا المخطط؟ + جدول العُقد + جدول الروابط + بلوك `diagram`.
types: `flowchart`، `bpmn`، `decision-tree`، `dfd`، `usecase`، `class`، `activity`. انظر SCHEMA.md v1.0 §Diagram.

**⚙️ الخطوات / الخوارزمية** — أسطر داخل fence ` ```algorithm ` بصيغة `1 | الخطوة | الأداة | ماذا يحدث` (سطر لكل خطوة). لا ترسم مربعات ولا أسهم — البارسر يحوّلها. انظر SCHEMA.md v1.0 §Algorithm.

pseudo-code في code fence بـ `language_tag=text`.

**الفهم الخاطئ ❌** / **الفهم الصحيح ✅** — سطر واحد لكل منهما.

#### مهم للامتحان ⚠️ / #### نقطة مهمة ⚠️ / #### ملاحظة: / #### الدرس المستفاد:
← `blockquote` يتبع كل منها مباشرةً.

**🤔 تفعيل الفهم** — استخدمه **≥3 مرات** في كل محاضرة.

كود ناقص — ضع `_______` أو `// (N)` مكان السطر الناقص.

تصحيح كود — قدّم الكود الخاطئ أولاً ثم المصحّح.

**Q{N}:** سؤال / **A:** إجابة.

**💡 التشبيه** — بعد كل مفهوم مجرّد: جملة واحدة من الحياة اليومية + «وجه الشبه: X = Y». استخدمه بكثرة. انظر SCHEMA.md v1.0 §Analogy.

**⚖️ المقايضة** — جدول: المزايا | العيوب | متى تختاره. للحالات التي لا يوجد فيها خطأ وصواب — لا تستخدم `compare` هنا. انظر SCHEMA.md v1.0 §Trade-off.

**🔄 قبل / بعد** — كود/حالة قبل العملية وبعدها + جملة «ماذا تغيّر؟». انظر SCHEMA.md v1.0 §Before-After.

**🔍 تتبع التنفيذ** (inline) — المدخل + جدول الخطوات (أعمدته تختلف حسب الموضوع) + النتيجة. انظر SCHEMA.md v1.0 §Trace.

**📐 المعادلة** — LaTeX داخل `$$` أو fence `math` (display_mode: both — استخدم أيضاً `$...$` للصيغ القصيرة داخل الفقرات). يتبعها **الشرح:** بمعنى كل رمز (`f(n)`، `g(n)`، `h(n)`، `C*`، `P*`، `α`، `β`، `θ`، `J(θ)`، `∇`، `∀`، `∃`، `⊢`، `⊨`). انظر SCHEMA.md v1.0 §Equation.
