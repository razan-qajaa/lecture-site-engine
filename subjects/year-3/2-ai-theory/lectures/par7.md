### 1. مراجعة: `Tree Search`

**النص الأصلي يقول:** الاستراتيجية تُعرَّف باختيار **ترتيب توسيع العقد** (`order of node expansion`).

**الشرح المبسّط:** كل خوارزميات البحث تعمل على نفس الهيكل الأساسي (`TREE-SEARCH`)، لكن الفرق الوحيد هو: **من أين نختار العقدة التالية التي نوسّعها؟** في `BFS` نأخذ من الأمام (قائمة انتظار)، في `DFS` نأخذ من الخلف (مكدس)، وفي البحث المُوجَّه نأخذ العقدة **الأفضل** وفق دالة تقييم.

#### 💻 الكود: `TREE-SEARCH`

#### ما هذا الكود؟
> الهيكل العام لكل خوارزميات البحث. يولّد العقد ويختبرها حتى يجد الحل أو يفشل.

```text
// General Tree Search Algorithm
function TREE-SEARCH(problem, fringe) returns solution or failure
  fringe ← INSERT(MAKE-NODE(INITIAL-STATE[problem]), fringe)  // initialize fringe
  loop do
    if fringe is empty then return failure         // no solution exists
    node ← REMOVE-FRONT(fringe)                   // pick next node by strategy
    if GOAL-TEST[problem] applied to STATE(node) succeeds
      return node                                  // solution found
    fringe ← INSERTALL(EXPAND(node, problem), fringe) // add children
```

#### شرح كل سطر:
1. `fringe ← INSERT(...)` → يُنشئ عقدة الحالة الابتدائية ويضعها في القائمة — نقطة البداية
2. `loop do` → يكرر حتى إيجاد حل أو فشل — القلب الدوّار للخوارزمية
3. `if fringe is empty` → إذا نفدت القائمة دون حل → فشل — شرط التوقف الأول
4. `node ← REMOVE-FRONT(fringe)` → يأخذ العقدة التالية **وفق الاستراتيجية** — هنا يكمن الفرق بين الخوارزميات
5. `if GOAL-TEST` → يختبر إن كانت العقدة هدفاً — شرط النجاح
6. `fringe ← INSERTALL(EXPAND(...))` → يُضيف أبناء العقدة المُوسَّعة → استمرار البحث

**الناتج المتوقع:**
> عقدة الهدف عند النجاح، أو `failure` إذا لم يوجد حل.

### 2. `Best-First Search` — البحث الأفضل-أولاً

**النص الأصلي يقول:** الفكرة: استخدام **دالة تقييم** (`evaluation function`) لكل عقدة — تقدير "مدى المرغوبية" (`desirability`) — ثم توسيع العقدة **الأكثر مرغوبية** غير المُوسَّعة.

**الشرح المبسّط:** بدلاً من البحث الأعمى، نُعطي كل عقدة "درجة" وندائماً نختار العقدة ذات أفضل درجة. كيف يُنفَّذ؟ `fringe` تكون **قائمة أولوية مرتّبة تنازلياً** حسب المرغوبية.

#### 💡 التشبيه:
> تخيّل أنك في مدينة غريبة وتبحث عن مطعم. بدلاً من التجوّل عشوائياً (`DFS/BFS`)، تفتح خرائط جوجل وتبدأ بأعلى المطاعم تقييماً.
> **وجه الشبه:** التقييم على الخريطة = دالة التقييم `evaluation function`.

| الحالة الخاصة | دالة التقييم |
|---|---|
| `Greedy Best-First Search` | `h(n)` فقط — التقدير إلى الهدف |
| `A* Search` | `f(n) = g(n) + h(n)` — الكلفة الفعلية + التقدير |

### 3. `Greedy Search` — البحث الجشع

**النص الأصلي يقول:** دالة التقييم `h(n)` (heuristic) = تقدير كلفة الوصول من `n` إلى أقرب هدف. مثال: `h_SLD(n)` = المسافة الهوائية من `n` إلى Bucharest. البحث الجشع يوسّع العقدة التي **تبدو** الأقرب للهدف.

**الشرح المبسّط:** الجشع يقول "أنا أهتم فقط بما تبقّى من المسافة للهدف، ولا يهمني كم قطعت". يختار دائماً العقدة التي تبدو أقرب للهدف بحسب التقدير `h(n)`، بغض النظر عن كلفة المسار الذي قطعه.

#### 📐 المعادلة: دالة الاستكشاف في `Greedy`

$$
f(n) = h(n)
$$

**الشرح:**
> - `h(n)`: القيمة الاستكشافية (`heuristic`) — تقدير التكلفة من العقدة `n` حتى الهدف
> - تُلاحَظ: لا يوجد `g(n)` هنا — التكلفة المقطوعة مُهمَلة تماماً

#### 🔍 تتبع التنفيذ: `Greedy Search` من Arad إلى Bucharest

**المدخل:** نقطة البداية = Arad، الهدف = Bucharest، القيم: `h_SLD` من الشريحة

| الخطوة | العقدة المُوسَّعة | `h(n)` | العقد في `fringe` | ملاحظة |
|---|---|---|---|---|
| 1 | Arad | 366 | {Sibiu=253, Timisoara=329, Zerind=374} | يختار Sibiu (أصغر h) |
| 2 | Sibiu | 253 | {Fagaras=176, Rimnicu=193, Arad=366, Oradea=380, Timisoara=329, Zerind=374} | يختار Fagaras (أصغر h) |
| 3 | Fagaras | 176 | {Bucharest=0, Sibiu=253, ...} | يختار Bucharest |
| 4 | Bucharest | 0 | — | **وصل الهدف!** ✅ |

**النتيجة:** Arad → Sibiu → Fagaras → Bucharest (كلفة = 140+99+211 = **450** كم)

#### ملاحظة:
> هذا ليس الحل الأمثل! الحل الأمثل عبر Rimnicu Vilcea يكلّف 418 كم.

### 4. خصائص `Greedy Search`

**النص الأصلي يقول:** تحليل الخصائص الأربع: `Complete?`، `Time?`، `Space?`، `Optimal?`

**الشرح المبسّط:** لنحلل كل خاصية بدقة:

#### مهم للامتحان ⚠️:
> جدول خصائص `Greedy Search`:

| الخاصية | النتيجة | السبب |
|---|---|---|
| **`Complete`** | ❌ **لا** | يمكن أن يدور في حلقات لا نهائية. مثال: إذا الهدف Oradea: Iasi→Neamt→Iasi→Neamt→... |
| **`Complete` (مع فحص التكرار)** | ✅ نعم في فضاء محدود | إذا أضفنا `repeated-state checking` |
| **`Time`** | `O(b^m)` | في أسوأ الحالات يفحص كل الفضاء، لكن `heuristic` جيد يُحسّن كثيراً |
| **`Space`** | `O(b^m)` | يحتفظ بكل العقد في الذاكرة |
| **`Optimal`** | ❌ **لا** | قد يجد حلاً غير أمثل كما رأينا في المثال |

#### 💡 التشبيه:
> الجشع مثل شخص يسير دائماً نحو الجبل الذي يبدو أقرب للقمة، لكنه قد يصل لـ"قمة محلية" (`local maximum`) وليس القمة الأعلى.
> **وجه الشبه:** القمة المحلية = حل غير أمثل | القمة الحقيقية = الحل الأمثل.

**الفهم الخاطئ الشائع ❌:** البحث الجشع يجد دائماً الحل الأقصر.
**الفهم الصحيح ✅:** يجد حلاً سريعاً لكنه ليس بالضرورة الأمثل — يتجاهل التكلفة المقطوعة.

### 5. `A* Search` — البحث المتميّز

**النص الأصلي يقول:** الفكرة: تجنّب توسيع المسارات التي هي بالفعل غالية الثمن. دالة التقييم `f(n) = g(n) + h(n)`. مع استخدام **heuristic مقبول** (`admissible`).

**الشرح المبسّط:** `A*` يجمع الأفضلين:
- `g(n)` = ما دفعته فعلاً حتى الآن (الكلفة الحقيقية)
- `h(n)` = تقدير ما سيكلّفك الوصول للهدف

بدلاً من الجشع الذي ينظر فقط للأمام أو `UCS` الذي ينظر فقط للخلف، `A*` ينظر في الاتجاهين.

#### 📐 المعادلة: `A* Evaluation Function`

$$
f(n) = g(n) + h(n)
$$

**الشرح:**
> - `g(n)`: التكلفة الفعلية من البداية حتى العقدة `n`
> - `h(n)`: التقدير الاستكشافي (`heuristic`) من `n` حتى الهدف
> - `f(n)`: التكلفة الإجمالية المُقدَّرة لأفضل مسار يمر بـ`n`
> - `h*(n)`: التكلفة الحقيقية من `n` للهدف (لا نعرفها، نُقدّرها بـ`h(n)`)

#### نقطة مهمة ⚠️:
> **شرط القبول (`Admissibility`):** يجب أن يكون `h(n) ≤ h*(n)` لكل `n`
> أي: `h(n)` لا يُبالغ أبداً في التقدير — يجب أن يكون **متفائلاً** دائماً.
> أيضاً يجب `h(n) ≥ 0` و `h(G) = 0` لأي هدف `G`.
> مثال: `h_SLD(n)` (المسافة الهوائية) لا تتجاوز المسافة الحقيقية على الطريق أبداً.

**الشرح المبسّط:** في كل خطوة نختار العقدة ذات أصغر `f(n)`.

#### 🔍 تتبع التنفيذ: `A* Search` من Arad إلى Bucharest

### 7. إثبات أمثلية `A*` — الطريقة المعيارية

**النص الأصلي يقول:** افرض أن هدفاً غير أمثل `G_2` وُلِّد وهو في القائمة. ليكن `n` عقدة غير مُوسَّعة على أقصر مسار للهدف الأمثل `G_1`. يمكن إثبات أن `f(G_2) > f(n)` لذا لن يختار `A*` توسيع `G_2` أبداً.

**الشرح المبسّط:** نثبت بالتناقض أن `A*` لا يمكنه اختيار هدف غير أمثل قبل الأمثل:

#### 📐 المعادلة: إثبات الأمثلية

$$
f(G_2) = g(G_2) \quad \text{لأن } h(G_2) = 0
$$

$$
g(G_2) > g(G_1) \quad \text{لأن } G_2 \text{ غير أمثل}
$$

$$
g(G_1) \geq f(n) \quad \text{لأن } h \text{ مقبول (admissible)}
$$

$$
\therefore f(G_2) > f(n)
$$

**الشرح:** بما أن `f(G_2) > f(n)`، سيختار `A*` دائماً `n` قبل `G_2`، وبالتالي سيجد المسار الأمثل عبر `n` قبل أن يصل لـ`G_2`. إذن `A*` **أمثل** (`Optimal`).

### 8. إثبات الأمثلية — الطريقة الأكثر فائدة (F-Contours)

**النص الأصلي يقول:** `Lemma`: يوسّع `A*` العقد بترتيب متزايد لقيم `f`. يُضيف تدريجياً "حدوداً f" (`f-contours`) مثلما يُضيف `BFS` طبقات. الحدّ `i` يحتوي كل العقد بـ `f = f_i` حيث `f_i < f_{i+1}`.

**الشرح المبسّط:** `A*` يعمل كدوائر تتسع تدريجياً — أولاً يُوسّع كل العقد ذات `f` أصغر، ثم يُكمل للدائرة التالية. هذا يُشبه `BFS` الذي يُوسّع طبقة كاملة قبل الانتقال لما يليها، لكن الدوائر هنا غير منتظمة الشكل.

#### 📊 المخطط: F-Contours في `A*`

#### ما هذا المخطط؟
> يُوضّح أن `A*` يُوسّع العقد دائرياً — كل دائرة تمثل قيمة `f` محددة، ويُكمل الدائرة الداخلية قبل الخارجية.

| # | العُقدة | النوع | الشرح |
|---|---|---|---|
| 1 | عقد الدائرة الداخلية | `explored` | `f = f_i` حيث `f_i` صغير |
| 2 | عقد الدائرة الخارجية | `frontier` | `f = f_{i+1} > f_i` |
| 3 | عقدة الهدف `G` | `goal` | تقع على حدّ `f = C*` |

### 9. خصائص `A*`

**النص الأصلي يقول:** تحليل الخصائص الأربع، مع قاعدة توسيع العقد حسب `C*`.

**الشرح المبسّط:** `A*` يُوسّع:
- **كل** العقد بـ `f(n) < C*`
- **بعض** العقد بـ `f(n) = C*`
- **لا شيء** من العقد بـ `f(n) > C*`

حيث `C*` هي كلفة الحل الأمثل.

#### مهم للامتحان ⚠️:
> جدول خصائص `A*` المقارن:

| الخاصية | النتيجة | التفصيل |
|---|---|---|
| **`Complete`** | ✅ **نعم** | إلا إذا كان هناك عدد لانهائي من العقد بـ `f ≤ f(G)` |
| **`Time`** | أسّي | أسّي في [الخطأ النسبي لـ`h` × طول الحل] |
| **`Space`** | `O(b^d)` | يحتفظ بكل العقد في الذاكرة — هذا هو المشكلة الرئيسية |
| **`Optimal`** | ✅ **نعم** | لا يمكن توسيع `f_{i+1}` قبل إنهاء `f_i` |

### 10. الاتساق (`Consistency`) — إثبات المبرهنة

**النص الأصلي يقول:** الـ`heuristic` متسق (`consistent`) إذا: `h(n) ≤ c(n, a, n') + h(n')` لكل عقدة `n` وخليفتها `n'` عبر إجراء `a`.

**الشرح المبسّط:** شرط الاتساق هو نسخة مشددة من الاستدراج. يقول: "التقدير من `n` لا يزيد على الخطوة الحقيقية لـ`n'` زائد التقدير من `n'`". هذا يُشبه عدم المساواة المثلثية.

#### 📐 المعادلة: الاتساق والتطبيق على `f`

$$
h(n) \leq c(n, a, n') + h(n')
$$

$$
f(n') = g(n') + h(n') = g(n) + c(n,a,n') + h(n') \geq g(n) + h(n) = f(n)
$$

**الشرح:**
> - `c(n, a, n')`: تكلفة الخطوة من `n` إلى `n'` عبر الإجراء `a`
> - النتيجة: `f(n')  ≥ f(n)` — أي `f(n)` لا يتناقص أبداً على أي مسار
> - هذا يُثبت المبرهنة: `A*` يُوسّع العقد بترتيب متزايد لـ`f`

#### 💡 التشبيه:
> الاتساق كعدم المساواة المثلثية في الهندسة: المسافة المباشرة من A إلى C لا تزيد على A→B + B→C. لا يُبالغ `h` في التقدير حتى للخطوات الوسيطة.

#### نقطة مهمة ⚠️:
> كل `heuristic` متسق هو بالضرورة مقبول (`admissible`)، لكن العكس ليس صحيحاً دائماً.

### 11. المقياس الاستكشافي المقبول (`Admissible Heuristics`) — لغز الـ8

**النص الأصلي يقول:** للغز الـ8-puzzle، هناك مقياسان مقبولان:
- `h_1(n)` = عدد القطع في غير موضعها الصحيح
- `h_2(n)` = مجموع مسافات `Manhattan` (عدد المربعات من كل قطعة لموضعها)

مع الحالة الابتدائية S: `h_1(S) = 6`، `h_2(S) = 4+0+3+3+1+0+2+1 = 14`

**الشرح المبسّط:** لغز الـ8 يوضّح كيف نختار `heuristic` مناسب:

#### 📐 المعادلة: حساب `h_2` لـ8-puzzle

$$
h_2(n) = \sum_{i=1}^{8} |x_i - x_i^{goal}| + |y_i - y_i^{goal}|
$$

**الشرح:**
> - `x_i, y_i`: موضع القطعة `i` الحالي
> - `x_i^{goal}, y_i^{goal}`: موضعها في الحالة الهدف
> - المجموع على كل القطع الـ8

| القطعة | موضعها الحالي | موضع الهدف | مسافة Manhattan |
|---|---|---|---|
| 1 | صف3 عمود3 | صف1 عمود1 | 2+2=4 |
| 2 | صف1 عمود2 | صف1 عمود2 | 0+0=0 |
| 3 | صف1 عمود3 | صف1 عمود3 | 0 (غير موجودة بشكل صحيح — 4-puzzle ≠ 8) |
| **المجموع** | — | — | **14** |

#### 🤔 تفعيل الفهم (اسأل نفسك):
> **سؤال:** لماذا `h_1` و `h_2` كلاهما **مقبولان** (`admissible`)؟
> **لأن:** في الواقع لا يمكن حل اللغز بعدد حركات أقل من عدد القطع في مكان خاطئ (`h_1`) ولا أقل من مجموع مسافات Manhattan (`h_2`). إذن كلاهما لا يُبالغ في التقدير.

### 12. الهيمنة (`Dominance`)

**النص الأصلي يقول:** إذا كان `h_2(n) ≥ h_1(n)` لكل `n` (وكلاهما مقبول)، فـ`h_2` يهيمن على `h_1` وهو أفضل للبحث. مقارنة تكاليف البحث:

| العمق | IDS | A*(h_1) | A*(h_2) |
|---|---|---|---|
| d=14 | 3,473,941 عقدة | 539 عقدة | 113 عقدة |
| d=24 | ~54,000,000,000 | 39,135 | 1,641 |

**الشرح المبسّط:** كلما كان `h(n)` أكبر (لكن ما زال مقبولاً)، كلما قلّت العقد المُوسَّعة. `h_2` يُقدّر قيماً أعلى من `h_1`، فيستبعد مسارات أكثر مبكراً.

#### 📐 المعادلة: دالة الهيمنة الجامعة

$$
h(n) = \max(h_a(n), h_b(n))
$$

**الشرح:**
> إذا كان لدينا أي مقياسين مقبولين `h_a` و `h_b`، فـأقصاهما مقبول أيضاً ويهيمن عليهما — الأفضل دائماً أخذ أعلى تقدير ممكن دون تجاوز الحقيقة.

#### مهم للامتحان ⚠️:
> `max(h_a, h_b)` مقبول دائماً إذا كان `h_a` و `h_b` مقبولَين. ويهيمن على كليهما.

### 13. المسائل المُرخَّصة (`Relaxed Problems`)

**النص الأصلي يقول:** يمكن اشتقاق المقاييس الاستكشافية المقبولة من **التكلفة الدقيقة لحل نسخة مُرخَّصة** من المسألة. إذا خُفِّفت قواعد لغز الـ8 بحيث تتحرك القطعة **لأي مكان** → `h_1`. إذا تحرّكت لأي مربع **مجاور** → `h_2`. النقطة الجوهرية: التكلفة المثلى للمسألة المُرخَّصة ≤ التكلفة المثلى للمسألة الحقيقية.

**الشرح المبسّط:** المسألة المُرخَّصة هي نسخة من المسألة الأصلية لكن بقواعد أقل تقييداً. الحل فيها دائماً أسهل (كلفة أقل أو مساوية)، لذا تكون كلفة الحل المثلى فيها حداً أدنى (`lower bound`) للكلفة في المسألة الأصلية — وهذا بالضبط شرط القبول!

#### 💡 التشبيه:
> المسألة المُرخَّصة كمسابقة الطائر للأماكن الحضرية، حيث تُحسَب المسافة "طائراً" وليس على الطريق. المسافة الهوائية دائماً ≤ مسافة الطريق الحقيقية.
> **وجه الشبه:** مسافة الطائر = `h(n)` | مسافة الطريق = `h*(n)`

#### مثال: مسألة البائع المتجوّل (`TSP`)

**النص الأصلي يقول:** مسألة TSP: أوجد أقصر جولة تزور كل المدن مرة واحدة. يمكن حساب **الشجرة الممتدة الصغرى** (`Minimum Spanning Tree`) في `O(n²)` وهي حد أدنى للجولة المفتوحة.

**الشرح المبسّط:** الـ`MST` هي نسخة مُرخَّصة من `TSP` — تتجاهل قيد "كل مدينة مرة واحدة فقط" والعودة للبداية. تكلفتها دائماً ≤ تكلفة `TSP`، فتصلح `heuristic` مقبول.

#### 📊 المخطط: `TSP` مقابل `MST`

#### ما هذا المخطط؟
> يُقارن بين رسم بياني `TSP` (الجولة الكاملة) والشجرة الممتدة الصغرى (الـ`heuristic`).

| # | العُقدة | النوع | الشرح |
|---|---|---|---|
| 1 | مدن المسألة | `state` | العقد في الرسم البياني |
| 2 | حواف TSP | `edge` | تمثل الرحلة الكاملة بين كل المدن |
| 3 | حواف MST | `edge` | نسخة مُرخَّصة — أشجار لا حلقات |

### تعريفات أساسية

| المصطلح | التعريف |
|---|---|
| `evaluation function` | دالة تُعطي لكل عقدة قيمة تمثل "مدى مرغوبيتها" |
| `heuristic h(n)` | تقدير التكلفة من العقدة `n` حتى أقرب هدف |
| `g(n)` | التكلفة الفعلية للوصول من البداية إلى `n` |
| `f(n) = g(n) + h(n)` | التكلفة الإجمالية المُقدَّرة عبر `n` |
| `admissible heuristic` | مقياس لا يُبالغ أبداً: `h(n) ≤ h*(n)` |
| `consistent heuristic` | `h(n) ≤ c(n,a,n') + h(n')` — اتسق مثلثي |
| `h*(n)` | التكلفة الحقيقية (الحقيقية) من `n` للهدف |
| `C*` | التكلفة المثلى للحل |
| `dominance` | `h_2 ≥ h_1` لكل `n` (كلاهما مقبول): `h_2` أفضل |
| `relaxed problem` | نسخة بقواعد أقل تقييداً — حلها يعطي `heuristic` مقبول |
| `f-contours` | "دوائر" العقد ذات نفس قيمة `f` — `A*` يُوسّعها بالترتيب |

### نقاط ذهبية

| # | النقطة |
|---|---|
| 1 | `Greedy` يُقلّل `h(n)` فقط — سريع لكن غير أمثل وغير كامل |
| 2 | `A*` يُقلّل `f(n) = g(n)+h(n)` — أمثل وكامل مع `h` مقبول |
| 3 | `h` مقبول شرط ضروري لأمثلية `A*` |
| 4 | `h` متسق شرط كافٍ لتوسيع `A*` بترتيب متزايد لـ`f` |
| 5 | كل `h` متسق هو مقبول (العكس ليس صحيحاً دائماً) |
| 6 | `max(h_a, h_b)` مقبول ويهيمن على كليهما |
| 7 | المسألة المُرخَّصة تُنتج مقاييس مقبولة تلقائياً |
| 8 | `A*` أكثر كفاءة من `IDS` والطرق الأخرى بفارق كبير جداً |

### أخطاء شائعة

| الخطأ | التصحيح |
|---|---|
| ظن أن `h` أي تقدير يكفي لـ`A*` | يجب أن يكون `h` مقبولاً (`admissible`): `h(n) ≤ h*(n)` |
| ظن أن `Greedy` أمثل | لا — يتجاهل `g(n)` وقد يجد حلاً غير أمثل |
| ظن أن `A*` لا يوسّع عقداً بعد الهدف | يُوسّع عقداً بـ `f < C*` حتى لو بعد اكتشاف الهدف |
| الخلط بين المقبول (`admissible`) والمتسق (`consistent`) | المتسق ⊂ المقبول، لكن ليس العكس |
| ظن أن `A*` الأسرع في الزمن | زمنه أسّي، لكن أقل عقداً من `BFS/IDS` بفضل الـ`heuristic` |

---

### خطوات الخوارزميات

#### ⚙️ الخطوات / الخوارزمية: `Best-First Search`

#### ما هدف هذه العملية؟
> توسيع العقدة الأفضل (الأعلى تقييماً) في كل خطوة. أساس `Greedy` و`A*`.

```algorithm
1 | تهيئة | أدخل الحالة الابتدائية في قائمة الأولوية (fringe) | fringe مرتّبة حسب f(n)
2 | اختبار الفراغ | إذا fringe فارغة → فشل | لا حل موجود
3 | اختيار | أخرج العقدة ذات أصغر f(n) | هذا هو جوهر البحث المُوجَّه
4 | اختبار الهدف | إذا كانت العقدة هدفاً → أعد الحل | النجاح
5 | التوسيع | أنتج أبناء العقدة وأدخلهم في fringe | استمرار البحث
6 | تكرار | ارجع لخطوة 2 | حلقة البحث
```

#### ⚙️ الخطوات / الخوارزمية: `A* Search`

#### ما هدف هذه العملية؟
> إيجاد الحل الأمثل بكفاءة باستخدام `f(n) = g(n) + h(n)` مع heuristic مقبول.

```algorithm
1 | تهيئة | أدخل الحالة الابتدائية: f=g+h في priority queue | g(start)=0، h(start) محسوب
2 | اختيار | أخرج العقدة n ذات أصغر f(n) | التركيز على المسارات الواعدة
3 | اختبار الهدف | إذا n هدف → الحل مثلى وأعده | بفضل admissibility
4 | التوسيع | لكل خليفة n' لـn | توليد الحالات الجديدة
5 | حساب g | g(n') = g(n) + c(n, a, n') | تراكم التكلفة الفعلية
6 | حساب h | h(n') = heuristic(n') | التقدير إلى الهدف
7 | حساب f | f(n') = g(n') + h(n') | التقييم الإجمالي
8 | إدراج/تحديث | أضف n' لـfringe إذا لم تُوسَّع، أو حدّث إذا وجدت مساراً أقصر | تحسين المسارات
9 | تكرار | ارجع لخطوة 2 | حتى الحل أو الفشل
```

#### نقاط التنفيذ:
- إذا وصلنا لعقدة عبر مسار أطول مسبقاً، يجب تحديث `g(n)` و`f(n)`
- عند وصول الهدف أول مرة في `TREE-SEARCH`، الحل أمثل بشرط `h` مقبول

## الجزء الثالث: أسئلة اختيار من متعدد (MCQ)

### السؤال 1 (medium): فرق أساسي

في `Best-First Search`، ما الذي يُحدد ترتيب توسيع العقد؟

أ) عمق العقدة في شجرة البحث
ب) دالة تقييم (`evaluation function`) لكل عقدة
ج) عدد الأبناء لكل عقدة
د) ترتيب إدراج العقدة في القائمة

**الإجابة: ب**
> أ) خاطئ — هذا `BFS`. ب) **صحيح** — جوهر `Best-First`. ج) خاطئ — عدد الأبناء لا علاقة له بالأولوية. د) خاطئ — هذا `BFS` أيضاً.

---

### السؤال 2 (medium): `Greedy` مقابل `A*`

ما الفرق الجوهري بين `Greedy Best-First` و`A*`؟

أ) `Greedy` يستخدم `h(n)` فقط، بينما `A*` يستخدم `g(n) + h(n)`
ب) `A*` أسرع دائماً من `Greedy`
ج) `Greedy` أمثل دائماً و`A*` ليس كذلك
د) `A*` لا يستخدم أي `heuristic`

**الإجابة: أ**
> أ) **صحيح** — الفرق الجوهري هو إضافة `g(n)`. ب) خاطئ — `Greedy` قد يكون أسرع لكن بحل أسوأ. ج) عكس الصحيح تماماً. د) خاطئ — `A*` يعتمد أساساً على `h(n)`.

---

### السؤال 3 (hard): خاصية الاكتمال

أي من التالي صحيح بشأن اكتمال (`completeness`) خوارزميتَي `Greedy` و`A*`؟

أ) كلاهما كاملتان في أي مسألة
ب) `Greedy` كاملة دائماً، `A*` قد تفشل
ج) `Greedy` غير كاملة (قد تدور في حلقات)، `A*` كاملة (إلا في حالات استثنائية)
د) كلاهما غير كاملتَين

**الإجابة: ج**
> أ) خاطئ — `Greedy` تدور في حلقات. ب) عكس الصحيح. ج) **صحيح** — `Greedy` قد تعلق: مثال Iasi→Neamt→Iasi. `A*` كاملة إلا إذا وُجد عدد لانهائي من العقد بـ`f ≤ f(G)`. د) خاطئ — `A*` كاملة في الغالب.

---

### السؤال 4 (hard): شرط الأمثلية

ما الشرط الضروري لأن يكون `A*` أمثلاً (`optimal`)؟

أ) أن يكون `h(n) ≥ h*(n)` (تُبالغ في التقدير)
ب) أن يكون `h(n) ≤ h*(n)` (مقبول — لا يُبالغ)
ج) أن يكون `h(n) = 0` دائماً
د) أن يكون `g(n) = 0` دائماً

**الإجابة: ب**
> أ) خاطئ — المبالغة تُفقد الأمثلية. ب) **صحيح** — الـ`admissibility` هو شرط الأمثلية. ج) خاطئ — `h=0` يُحوّل `A*` لـ`UCS` (صحيح لكن غير كفؤ). د) خاطئ — `g=0` يُحوّل `A*` لـ`Greedy` وهو غير أمثل.

### السؤال 5 (hard): تتبع A*

في شجرة بحث `A*` بالمعطيات التالية: A(g=0,h=10), B(g=5,h=6), C(g=7,h=4), D(g=10,h=0). أي العقد تُوسَّع أولاً؟

أ) A (f=10)
ب) C (f=11)
ج) B (f=11)
د) D (f=10)

**الإجابة: أ**
> f(A)=0+10=10, f(B)=5+6=11, f(C)=7+4=11, f(D)=10+0=10. A وD كلاهما f=10. نبدأ بـA (الحالة الابتدائية). إذا كانت D هدفاً وf=10=C*، فسيُوسَّع بعد A. الإجابة أ هي الصحيحة.

### السؤال 6 (medium): خاصية التكلفة

`A*` على مسألة لغز الـ8. أي المقياسَين أفضل للبحث وأقل عقداً مُوسَّعة؟

أ) `h_1 = 0` (لا تقدير)
ب) `h_1` = عدد القطع في غير مكانها
ج) `h_2` = مجموع مسافات Manhattan
د) `h_3` = عدد القطع ÷ 2

**الإجابة: ج**
> أ) `h=0` يُحوّل `A*` لـ`UCS` — الأسوأ. ب) مقبول لكن ليس الأفضل. ج) **صحيح** — `h_2 ≥ h_1` لأن Manhattan أدق. د) `h_3 < h_1 ≤ h_2`، أسوأ من الثلاثة.

### السؤال 7 (hard): الاتساق

ما النتيجة المباشرة لكون `h(n)` متسقاً (`consistent`)؟

أ) `f(n)` يتناقص على طول أي مسار
ب) `f(n)` لا يتناقص أبداً على طول أي مسار
ج) `h(n) = h*(n)` دائماً
د) `A*` يُوسّع عقداً بترتيب عشوائي

**الإجابة: ب**
> أ) خاطئ — الاتساق يضمن عدم التناقص لا التناقص. ب) **صحيح** — من `f(n') ≥ f(n)` يتبع أن `f` غير متناقصة. ج) خاطئ — `h` تقدير وليس القيمة الحقيقية. د) خاطئ — الاتساق يجعل `A*` يُوسّع بترتيب متزايد لـ`f`.

### السؤال 8 (hard): الهيمنة

إذا كان `h_a(n) ≤ h_b(n)` لكل `n` وكلاهما مقبول، ما الصحيح؟

أ) `h_a` يُوسّع عقداً أقل من `h_b`
ب) `h_b` يُوسّع عقداً أقل من `h_a` وهو أكثر كفاءة
ج) كلاهما يُوسّع نفس العدد من العقد
د) `h_b` غير مقبول إذا كان أكبر من `h_a`

**الإجابة: ب**
> `h_b ≥ h_a` أي `h_b` يُقدّر قيماً أعلى → يستبعد مسارات أكثر مبكراً → أقل عقداً. أ) عكس الصحيح. ب) **صحيح**. ج) خاطئ. د) خاطئ — الهيمنة لا تنقض القبول.

---

### السؤال 9 (medium): المسألة المُرخَّصة

لماذا تُنتج المسألة المُرخَّصة (`relaxed problem`) مقياساً مقبولاً (`admissible heuristic`)?

أ) لأن حلها دائماً أصعب من الأصلية
ب) لأن كلفة حلها دائماً أقل من أو تساوي كلفة الحل الأصلي
ج) لأنها تستخدم نفس القواعد بالضبط
د) لأنها تجاهل `g(n)` دائماً

**الإجابة: ب**
> أ) خاطئ — أسهل بتخفيف القيود. ب) **صحيح** — كلفة الحل المثلى في النسخة المُرخَّصة ≤ الأصلية، وهذا هو تعريف القبول. ج) خاطئ — تُرخِّص أي تُخفّف القواعد. د) خاطئ.

### السؤال 10 (hard): إثبات أمثلية A*

في إثبات أمثلية `A*`، لماذا لا يُوسَّع الهدف الغير أمثل `G_2` قبل الأمثل `G_1`؟

أ) لأن `f(G_2) = f(G_1)` دائماً
ب) لأن `f(G_2) > f(n)` حيث `n` عقدة على مسار `G_1`، وبالتالي `A*` يُوسّع `n` أولاً
ج) لأن `G_2` لا يُضاف لـ`fringe` أبداً
د) لأن `g(G_2) = 0`

**الإجابة: ب**
> أ) خاطئ — `f(G_2) > f(G_1)`. ب) **صحيح** — `f(G_2) = g(G_2) > g(G_1) ≥ f(n)` فيُوسَّع `n` أولاً. ج) خاطئ — قد يُضاف. د) خاطئ — `g(G_2)` هي التكلفة الحقيقية للوصول إليه.

### السؤال 11 (medium): خصائص Greedy

أي التالي **صحيح** بشأن `Greedy Best-First`؟

أ) دائماً يجد الحل الأمثل إذا وُجد
ب) كلفته الزمنية O(b^m) ويحتفظ بكل العقد في الذاكرة
ج) لا يُمكن أن يدور في حلقات
د) يستخدم `g(n)` فقط بدون `h(n)`

**الإجابة: ب**
> أ) خاطئ — غير أمثل. ب) **صحيح** — `O(b^m)` زمناً وفضاءً. ج) خاطئ — يدور في حلقات مثل Iasi-Neamt. د) خاطئ — يستخدم `h(n)` فقط.

### السؤال 12 (hard): h=0 في A*

إذا استخدمنا `h(n) = 0` (heuristic خالي) في `A*`، بأي خوارزمية يتصرف؟

أ) `DFS`
ب) `BFS`
ج) `Uniform Cost Search (UCS)`
د) `Greedy Search`

**الإجابة: ج**
> عند `h=0`، `f(n) = g(n) + 0 = g(n)` → `A*` يُوسَّع بترتيب `g(n)` متصاعد = `UCS`. أ) خاطئ. ب) خاطئ (BFS يُوسَّع بالعمق). ج) **صحيح**. د) خاطئ.

### السيناريو 1: تتبع `A*` على رسم بياني صغير

> رسم بياني: S→A (تكلفة 1)، S→B (تكلفة 4)، A→C (تكلفة 2)، B→C (تكلفة 1)، C→G (تكلفة 3)
> قيم h: h(S)=6، h(A)=4، h(B)=2، h(C)=2، h(G)=0

### السؤال 1.1 (hard): أي عقدة تُوسَّع أولى بعد S؟

أ) A: f=1+4=5
ب) B: f=4+2=6
ج) G: f=0

### سؤال تصحيح 1 (wrong_heuristic): heuristic غير مقبول

**الكود الخاطئ:**
```text
// A* with wrong heuristic
function A_STAR(problem):
  h(n) = STRAIGHT_LINE_DISTANCE(n, goal) * 2   // ← الخطأ هنا
  f(n) = g(n) + h(n)
  return BEST_FIRST_SEARCH(problem, f)
```

**اكتشف الخطأ:** تضعيف المسافة الهوائية يجعل `h(n) > h*(n)` في بعض الحالات.

**التصحيح:**
```text
h(n) = STRAIGHT_LINE_DISTANCE(n, goal)   // ← لا تضاعف — h_SLD مقبول أصلاً
```

**شرح الحل:**
1. `h_SLD` مقبول لأنه لا يُبالغ — المسافة الهوائية ≤ مسافة الطريق
2. ضربها في 2 قد يجعلها تتجاوز المسافة الحقيقية → يفقد القبول (`admissibility`)
3. بدون القبول، لا يمكن ضمان أمثلية `A*`

### سؤال تصحيح 2 (infinite_loop): حلقة لا نهائية في Greedy

**الكود الخاطئ:**
```text
// Greedy Search without loop detection
function GREEDY_SEARCH(problem):
  fringe = MAKE_QUEUE(INITIAL_STATE)
  loop:
    node = REMOVE_BEST(fringe, h)    // expand best h(n)
    if GOAL_TEST(node): return node
    fringe = INSERT_ALL(EXPAND(node), fringe)   // ← لا فحص للتكرار!
```

**اكتشف الخطأ:** لا يفحص الحالات المُزارة → يدور في حلقة لا نهائية.
مثال: Iasi→Neamt→Iasi→Neamt→...

**التصحيح:**
```text
function GREEDY_SEARCH(problem):
  fringe = MAKE_QUEUE(INITIAL_STATE)
  closed = EMPTY_SET()               // ← أضف مجموعة الحالات المُزارة
  loop:
    if EMPTY(fringe): return failure
    node = REMOVE_BEST(fringe, h)
    if GOAL_TEST(node): return node
    if node.state NOT IN closed:     // ← فحص التكرار
      closed.ADD(node.state)
      fringe = INSERT_ALL(EXPAND(node), fringe)
```

**شرح الحل:**
1. `closed set` يحفظ كل الحالات التي وُسِّعت مسبقاً
2. نفحص قبل التوسيع هل العقدة في `closed`
3. هذا يجعل `Greedy` كاملاً في الفضاء المحدود

### سؤال تصحيح 3 (logic): خطأ في تحديث `g(n)` في A*

**الكود الخاطئ:**
```text
// A* with wrong g update
for each successor n' of n:
  new_g = g(n) + cost(n, n')
  if n' NOT in fringe and n' NOT in closed:
    g(n') = new_g
    fringe.INSERT(n', f(n'))
  // ← ينسى تحديث العقدة إذا وجد مساراً أقصر!
```

**اكتشف الخطأ:** إذا وُجدت `n'` في `fringe` بـ`g` أعلى (مسار أطول)، يجب تحديثها بالمسار الأقصر.

**التصحيح:**
```text
for each successor n' of n:
  new_g = g(n) + cost(n, n')
  if n' NOT in fringe and n' NOT in closed:
    g(n') = new_g
    fringe.INSERT(n', g(n')+h(n'))
  elif new_g < g(n'):               // ← إذا وجدنا مساراً أقصر
    g(n') = new_g                   // ← حدّث g
    fringe.UPDATE(n', g(n')+h(n'))  // ← وحدّث f في الـfringe
```

**شرح الحل:**
1. قد تُكتشف `n'` عبر مسار أول غير أمثل
2. لاحقاً قد نجد طريقاً أقصر لـ`n'` → يجب تحديث `g` و`f`
3. بدون هذا، قد يجد `A*` حلاً غير أمثل

### سؤال تصحيح 4 (misconception): خطأ في اختبار الهدف

**الكود الخاطئ:**
```text
// Wrong: testing goal when generating, not expanding
for each successor n' of n:
  if GOAL_TEST(n'): return n'        // ← فحص عند التوليد لا التوسيع!
  fringe.INSERT(n')
```

**اكتشف الخطأ:** في `A*`، يجب فحص الهدف عند **توسيع** العقدة (إزالتها من fringe)، لا عند توليدها.

**التصحيح:**
```text
// Correct: test goal when expanding (removing from fringe)
node = fringe.REMOVE_MIN()
if GOAL_TEST(node): return node      // ← فحص عند التوسيع
for each successor n' of node:
  fringe.INSERT(n')                  // ← فقط أضف، لا تختبر
```

**شرح الحل:**
1. الفحص عند التوليد قد يعيد حلاً غير أمثل
2. لأن العقدة حين تُولَّد لم يُتحقَّق بعد أنها مسار أمثل
3. عند التوسيع (إزالتها من fringe)، نضمن أنها وصلت بأمثل مسار

         5 _ 6              4 5 6
         8 3 1              7 8 _

h_1: القطع في غير مكانها: {7,5,6,8,3,1} = 6
h_2: Manhattan:
  7: من(0,0) إلى(2,0) = 2+0 = 2
  2: صحيح = 0
  4: من(0,2) إلى(1,0) = 1+2 = 3

## الجزء الرابع: بطاقات Q&A (≥16 بطاقة)

**Q1:** ما الفرق بين `h(n)` و `h*(n)`؟
**A:** `h(n)` تقدير التكلفة (قد يخطئ)، `h*(n)` التكلفة الحقيقية الفعلية للهدف من `n`.

---

**Q2:** متى يكون `A*` أمثلاً؟
**A:** عندما يكون الـ`heuristic` مقبولاً: `h(n) ≤ h*(n)` لكل `n`.

---

**Q3:** ما الفرق بين القبول (`admissibility`) والاتساق (`consistency`)؟
**A:** القبول: `h(n) ≤ h*(n)`. الاتساق: `h(n) ≤ c(n,a,n') + h(n')`. كل متسق مقبول، ليس العكس.

---

**Q4:** لماذا `Greedy` غير أمثل؟
**A:** لأنه يتجاهل `g(n)` (التكلفة المقطوعة) ويختار فقط بحسب `h(n)`، فقد يختار مساراً يبدو قريباً لكنه غالٍ.

---

**Q5:** ما النتيجة العملية للهيمنة في الـ`heuristics`؟
**A:** المقياس المهيمن يُوسَّع معه عقداً أقل وبالتالي البحث أسرع مع ضمان نفس الأمثلية.

---

**Q6:** ما `h_SLD` ولماذا هو مقبول في خريطة رومانيا؟
**A:** المسافة الهوائية بين المدن. مقبول لأن المسافة الهوائية ≤ مسافة الطريق الحقيقية دائماً.

---

**Q7:** ما معنى `f-contours` في `A*`؟
**A:** مجموعات العقد التي لها نفس قيمة `f`. `A*` يُكمل كل "حدّ f" قبل الانتقال للتالي.

---

**Q8:** إذا استخدمنا `h(n) = h*(n)` (القيمة الحقيقية)، كيف يتصرف `A*`؟
**A:** يُوسّع أقل عدد ممكن من العقد — يتبع المسار الأمثل مباشرة. لكن حساب `h*` مكلف للغاية.

---

**Q9:** ما الخوارزمية التي تنتج `A*` عند `h(n)=0`؟
**A:** `Uniform Cost Search (UCS)` — يُوسّع بترتيب `g(n)` متصاعد.

---

**Q10:** هل `max(h_1, h_2)` مقبول إذا كان `h_1` و`h_2` مقبولَين؟
**A:** نعم، دائماً. `max ≤ h*` لأن كلاهما `≤ h*`، والـ`max` يهيمن على كليهما.

---

**Q11:** ما خاصية `A*` التي تجعل استخدامه في الذاكرة مشكلة؟
**A:** `A*` يحتفظ بكل العقد المُولَّدة في الذاكرة (`Space = O(b^d)`) — قد يستنزف RAM.

---

**Q12:** ما المسألة المُرخَّصة (`relaxed problem`) وكيف تُفيد في تصميم `heuristics`؟
**A:** نسخة من المسألة بقواعد أقل تقييداً. حلها المثلى ≤ حل الأصلية → يُعطي `heuristic` مقبولاً تلقائياً.

---

**Q13:** لماذا تُفيد `MST` كـ`heuristic` لـ`TSP`؟
**A:** لأن `MST` هي حل مسألة مُرخَّصة من `TSP` (بدون شرط "مرة واحدة")، وتكلفتها ≤ تكلفة `TSP`.

---

**Q14:** ما الفرق الرئيسي بين `BFS` و`Best-First Search`؟
**A:** `BFS` يُوسَّع بترتيب العمق (FIFO). `Best-First` يُوسَّع بترتيب دالة التقييم `f(n)`.

---

**Q15:** ما هو `h_1` و`h_2` في لغز الـ8؟
**A:** `h_1` = عدد القطع في غير مكانها. `h_2` = مجموع مسافات Manhattan. كلاهما مقبول، `h_2` يهيمن.

---

**Q16:** متى يكون `Greedy` كاملاً؟
**A:** في الفضاء المحدود مع فحص التكرار (`repeated-state checking`).

---

**Q17:** أيهما يُوسَّع عقداً أكثر: `A*(h_1)` أم `A*(h_2)` في لغز الـ8؟
**A:** `A*(h_1)` يُوسَّع عقداً أكثر — لأن `h_2` يهيمن على `h_1` ويستبعد مسارات أكثر.

---

**Q18:** ما الإثبات الذي يستخدمه `A*` لضمان عدم اختيار هدف غير أمثل؟
**A:** `f(G_2) = g(G_2) > g(G_1) ≥ f(n)` حيث `n` على مسار `G_1`، لذا `A*` يُوسَّع `n` قبل `G_2`.

```text
// ===================================================
// GREEDY BEST-FIRST SEARCH
// ===================================================
function GREEDY_SEARCH(problem, h) returns solution or failure
  // h(n) = heuristic estimate of cost from n to goal
  fringe ← MAKE_PRIORITY_QUEUE([MAKE_NODE(INITIAL_STATE[problem])], key=h)
  closed ← EMPTY_SET()
  loop do
    if EMPTY(fringe) then return failure
    node ← REMOVE_MIN(fringe)          // node with smallest h(n)
    if GOAL_TEST[problem](STATE(node)) then return SOLUTION(node)
    if STATE(node) not in closed then
      closed ← INSERT(STATE(node), closed)
      for each child in EXPAND(node, problem) do
        fringe ← INSERT(child, fringe)  // ordered by h
  end loop
```

```text
// ===================================================
// A* SEARCH
// ===================================================
function A_STAR_SEARCH(problem, h) returns solution or failure
  // f(n) = g(n) + h(n); h must be admissible
  start ← MAKE_NODE(INITIAL_STATE[problem])
  start.g ← 0
  start.f ← h(INITIAL_STATE[problem])
  fringe ← MAKE_PRIORITY_QUEUE([start], key=f)
  closed ← EMPTY_SET()
  loop do
    if EMPTY(fringe) then return failure
    node ← REMOVE_MIN(fringe)             // node with smallest f(n)
    if GOAL_TEST[problem](STATE(node)) then return SOLUTION(node)
    if STATE(node) not in closed then
      closed ← INSERT(STATE(node), closed)
      for each (action, child_state, step_cost) in SUCCESSORS(node) do
        child ← MAKE_NODE(child_state)
        child.g ← node.g + step_cost
        child.h ← h(child_state)
        child.f ← child.g + child.h
        if child_state not in closed then
          if child_state in fringe with higher f then
            UPDATE(fringe, child)          // found shorter path: update
          else
            INSERT(fringe, child)
  end loop
```

```text
// ===================================================
// BEST-FIRST SEARCH (General Template)
// ===================================================
function BEST_FIRST_SEARCH(problem, f) returns solution or failure
  // f = evaluation function (h for Greedy, g+h for A*)
  fringe ← MAKE_PRIORITY_QUEUE([MAKE_NODE(INITIAL_STATE[problem])], key=f)
  loop do
    if EMPTY(fringe) then return failure
    node ← REMOVE_MIN(fringe)
    if GOAL_TEST[problem](STATE(node)) then return node
    fringe ← INSERTALL(EXPAND(node, problem), fringe)
  end loop
```

```text
// ===================================================
// ADMISSIBILITY CHECK (Conceptual)
// ===================================================
function IS_ADMISSIBLE(h, problem) returns boolean
  // Check: h(n) ≤ h*(n) for all n
  for each state n in STATE_SPACE(problem) do
    true_cost ← OPTIMAL_COST_TO_GOAL(n, problem)  // h*(n) — expensive!
    if h(n) > true_cost then return false
  return true

// ===================================================
// CONSISTENCY CHECK (Conceptual)
// ===================================================
function IS_CONSISTENT(h, problem) returns boolean
  // Check: h(n) ≤ c(n,a,n') + h(n') for all edges
  for each state n in STATE_SPACE(problem) do
    for each (action, n', step_cost) in SUCCESSORS(n) do
      if h(n) > step_cost + h(n') then return false
  return true
```

## الجزء الخامس: أسئلة نظرية متوقعة بالامتحان

---

### السؤال 1: عرّف `Best-First Search` وكيف يختلف عن `BFS` و `DFS`؟

**نموذج الإجابة:**
1. **التعريف:** خوارزمية بحث تستخدم دالة تقييم `f(n)` لكل عقدة وتُوسّع دائماً العقدة ذات أصغر (أو أكبر) قيمة.
2. **المكونات:** دالة التقييم `f(n)`، قائمة أولوية (`priority queue`) للـ`fringe`
3. **مثال:** `Greedy` يستخدم `f=h(n)`، `A*` يستخدم `f=g(n)+h(n)`
4. **متى نستخدم:** عندما لدينا معرفة مسبقة (`heuristic`) عن المسألة تُساعد على تقدير المسافة للهدف

الفرق: `BFS` يُوسّع بعمق (FIFO)، `DFS` يُوسّع بعمق (LIFO)، `Best-First` يُوسّع بقيمة `f`.

---

### السؤال 2: ما المقياس الاستكشافي المقبول (`admissible heuristic`)؟ لماذا هو ضروري لـ`A*`؟

**نموذج الإجابة:**
1. **التعريف:** `h(n) ≤ h*(n)` لكل عقدة `n` — لا يُبالغ في التقدير أبداً
2. **المكونات:** `h(n)` التقدير، `h*(n)` التكلفة الحقيقية، شرط `h(n) ≥ 0`، `h(G)=0`
3. **مثال:** المسافة الهوائية `h_SLD` مقبولة، Manhattan مقبول للـ8-puzzle
4. **متى نستخدم:** ضروري لضمان أمثلية `A*` — بدونه قد يتجاهل `A*` المسار الأمثل

---

### السؤال 3: ما خصائص `A*` الأربع؟ ودليل الأمثلية بالإثبات المعياري؟

**نموذج الإجابة:**
1. **التعريف:** `A* Complete?`: نعم (إلا ∞ عقد). `Time?`: أسي. `Space?`: O(b^d). `Optimal?`: نعم مع admissible h
2. **الإثبات:** `f(G_2)=g(G_2)>g(G_1)≥f(n)` حيث `n` على مسار `G_1` → `A*` يُوسَّع `n` قبل `G_2`
3. **مثال:** رومانيا — `A*` يجد 418 لا 450
4. **متى:** `A*` مفضل حين يوجد `h` مقبول جيد ونريد ضمان الأمثلية

---

### السؤال 4: ما الاتساق (`consistency`)؟ كيف يختلف عن القبول؟

**نموذج الإجابة:**
1. **التعريف:** `h(n) ≤ c(n,a,n') + h(n')` — شرط مثلثي
2. **المكونات:** `c(n,a,n')` تكلفة الخطوة، النتيجة: `f(n)` غير متناقصة
3. **الفرق:** كل متسق مقبول. المقبول ليس بالضرورة متسقاً. المتسق يضمن توسيعاً مرتباً
4. **متى نستخدم:** الاتساق يُبسَّط تنفيذ `A*` — لا نحتاج لتحديث العقد في `closed`

---

### السؤال 5: ما الهيمنة (`dominance`) في المقاييس؟ ما تأثيرها على كفاءة البحث؟

**نموذج الإجابة:**
1. **التعريف:** `h_2` يهيمن على `h_1` إذا كان `h_2(n) ≥ h_1(n)` لكل `n` (كلاهما مقبول)
2. **التأثير:** `A*(h_2)` يُوسَّع عقداً أقل من `A*(h_1)` → أسرع
3. **مثال:** في d=24: `A*(h_2)` = 1,641 عقدة مقابل `A*(h_1)` = 39,135
4. **متى:** استخدم دائماً أعلى `h` مقبول متاح، أو `max(h_a, h_b)` إذا توفر عدة مقاييس

---

### السؤال 6: ما المسألة المُرخَّصة؟ كيف تُنتج مقياساً مقبولاً؟

**نموذج الإجابة:**
1. **التعريف:** نسخة من المسألة الأصلية بقواعد مُخفَّفة (أقل تقييداً)
2. **الآلية:** التكلفة المثلى للمسألة المُرخَّصة ≤ الأصلية → حدّ أدنى → مقبول تلقائياً
3. **مثال:** 8-puzzle مُرخَّص "تحرك لأي مكان" → `h_1`. "تحرك لأي مجاور" → `h_2`
4. **متى:** طريقة منهجية لتصميم `heuristics` مضمونة القبول

---

### السؤال 7: لماذا تُعدّ الذاكرة المشكلة الرئيسية لـ`A*`؟

**نموذج الإجابة:**
1. **التعريف:** `A*` يحتفظ بجميع العقد المُولَّدة في الذاكرة
2. **المكونات:** `fringe` + `closed` تنمو بـ`O(b^d)`
3. **مثال:** في d=24 للـ8-puzzle، حتى لو فحصنا 1,641 فقط، نخزن كل المسارات
4. **الحل:** `IDA*` (A* مع iterative deepening) يُقلّل الذاكرة لـ`O(d)` على حساب الزمن

---

### السؤال 8: ما الفرق بين `Greedy` و`A*` من حيث الأمثلية والاكتمال؟

**نموذج الإجابة:**
1. **الأمثلية:** `Greedy` — لا أمثلية. `A*` — أمثل مع `h` مقبول
2. **الاكتمال:** `Greedy` — غير كامل (حلقات). `A*` — كامل في الغالب
3. **مثال:** رومانيا — `Greedy` يجد 450 كم، `A*` يجد 418 كم
4. **السبب:** `Greedy` يتجاهل `g(n)` → لا يرى التكلفة المتراكمة

---

### السؤال 9: ما الـ`f-contours` وكيف تثبت أمثلية `A*`؟

**نموذج الإجابة:**
1. **التعريف:** مجموعات العقد بنفس قيمة `f` — `A*` يُكمل كل مجموعة قبل التالية
2. **الآلية:** لا يُوسَّع `f_{i+1}` حتى تُوسَّع كل عقد `f_i` → عند وصول الهدف، كل مسارات أقل تكلفة جُرِّبت
3. **النتيجة:** الحل المُوجَد يكون دائماً على حدّ `f = C*` وهو الأمثل
4. **متى:** الـ`f-contours` تعني أن `A*` لا "يقفز" لمسارات غالية قبل رخيصة

---

### السؤال 10: هل `h_2(n) = total Manhattan distance` متسق لـ8-puzzle؟

**نموذج الإجابة:**
1. **التعريف:** يجب التحقق من `h(n) ≤ c(n,a,n') + h(n')` لكل خطوة
2. **الإثبات:** كل خطوة في اللغز تُغيّر مسافة Manhattan لقطعة واحدة بـ±1. لذا `h(n) - h(n') ≤ 1 = c` → `h(n) ≤ c + h(n')` ✓
3. **مثال:** تحريك قطعة للخلف يزيد `h` بـ1، للأمام يُنقصه بـ1، كلا الحالتَين يُحقق الاتساق
4. **النتيجة:** `h_2` متسق → `f(n)` غير متناقصة → `A*` يعمل بكفاءة أعلى

### 🔑 أهم النقاط الذهبية

| الموضوع | النقطة |
|---|---|
| `Best-First` | يُوسَّع بـ`evaluation function` — جوهر `Greedy` و`A*` |
| `Greedy` | `f=h(n)` — سريع لكن غير أمثل وغير كامل |
| `A*` | `f=g+h` — أمثل وكامل مع `h` مقبول |
| `Admissible` | `h(n) ≤ h*(n)` — شرط أمثلية `A*` |
| `Consistent` | `h(n) ≤ c+h(n')` — يضمن `f` غير متناقصة |
| `Dominance` | `h_2≥h_1` → `A*(h_2)` أسرع وأكفأ |
| `Relaxed` | نسخة مُخففة → تُعطي `h` مقبولاً تلقائياً |

### 🔑 مرجع سريع

| الرمز | المعنى | يُستخدم في |
|---|---|---|
| `h(n)` | heuristic — تقدير التكلفة للهدف | Greedy, A* |
| `g(n)` | التكلفة الفعلية من البداية لـ`n` | UCS, A* |
| `f(n)` | `g(n)+h(n)` — تقدير الكلفة الكلية | A* |
| `h*(n)` | التكلفة الحقيقية من `n` للهدف | تعريف القبول |
| `C*` | كلفة الحل الأمثل | تحليل A* |
| `h_SLD` | المسافة الهوائية | مثال رومانيا |
| `h_1` | عدد القطع خاطئة الموضع | 8-puzzle |
| `h_2` | مجموع Manhattan distances | 8-puzzle (أفضل) |
