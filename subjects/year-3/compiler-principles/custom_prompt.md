## طبيعة المادة

| النوع | الاستخدام | أمثلة |
| --- | --- | --- |
| نظري | تعريفات ومبادئ | `CFG`، `Ambiguity`، `AST` |

## القواعد الإلزامية

- لا تتجاهل أي سطر أو معلومة وردت في المحاضرة
- أكمل الناقص مع وسم **(شرح زيادة للفهم)** أو **(غير مشروحة في المحاضرة)**
- ابدأ من المبتدئ، لا تنتقل لنقطة قبل إتمام شرح السابقة
- اشرح **لماذا** وراء كل فكرة، لا التعريف فقط
- تشبيه يومي + مثال عملي بعد كل نقطة
- اتبع تسلسل المحاضرة نفسها
- حلّل نوع المحاضرة قبل البدء وطبّق القالب المناسب

---

## التكيف مع نوع المحاضرة

قبل الشرح، حدّد فئة المحاضرة وطبّق القالب المناسب في الجزء الأول:

- إذا كانت عن **Lexical Analysis** → استخدم `Regular Expressions`، `DFA`، `NFA`، `Scanner`
- إذا كانت عن **Syntax Analysis** → استخدم `CFG`، `Derivation`، `Parse Tree`، `Ambiguity`
- إذا كانت عن **Parsing** → استخدم `Top-Down`، `Bottom-Up`، `LR Parser`، `SLR`
- إذا كانت عن **Code Generation** → استخدم `AST`، `Stack Machine`، `Register Machine`، `Optimization`

## الجزء الثاني: ملخص منظم

## 🎯 أنواع أسئلة إضافية إلزامية للأتمتة (الفحص بالكامل آلي)

> هذه الأنواع تُدمَج **ضمن** الأجزاء أعلاه (لا تُنشئ لها أقساماً ## منفصلة) — التزم بها لضمان كفاية وتنوّع الأسئلة القابلة للتصحيح الآلي.

### مراجعة سريعة (سؤال / جواب) — ضمن qa_cards / qa_inline / theory (≥16، easy)
**Q{N}:** سؤال مراجعة مباشر (تعريف، فرق، متى نستخدم) — بدون تعليل طويل.
**A:** إجابة مختصرة قابلة للمقارنة الآلية. مثال: «ما الفرق بين `First` و `Follow`؟»

### تحدي مركّب (سيناريو + أسئلة متعددة) — ضمن mcq / debug / theory (≥4، hard)
### السيناريو {N}: {عنوان}
> [قواعد نحوية، جدول تحليل، أو اشتقاق كامل — 5–10 أسطر]
ثم 3–5 أسئلة MCQ **كلها مبنية على نفس السيناريو**. كل سيناريو يختبر: فهم + تطبيق + قرار.
المصادر: `grammar_definition`، `parse_table_state`، `lexical_input_string`، `ast_construction`.
مثال: «جدول تحليل LR ناقص — أي إجراء في الحالة X مع الرمز Y؟»

### تطبيق عملي (أكمل / صحّح / نفّذ) — ضمن exercise / analysis_exercise / debug (≥6، medium)
كل تمرين: **المعطيات** + **المطلوب** (مرقّم) + **نموذج الحل الكامل**. الإجابة قابلة للمقارنة حرفياً أو رقمياً.
الأنواع: `first_follow_calc`، `grammar_transform`، `table_fill`، `parse_table_construction`، `diagram_completion`.

### تتبع خطوة بخطوة (جدول ناقص) — ضمن trace_exercise / mcq (≥5، medium)
**المدخل:** {سلسلة رموز أو قواعد نحوية} → **أكمل الجدول** → **نموذج الحل** + سؤال MCQ على نفس التتبع.
الأهداف: `parse_stack_trace`، `lr_table_construction`، `derivation_steps`، `dfa_nfa_simulation`.

### مقارنة وتحليل — ضمن mcq / theory / analysis_exercise (≥4، medium)
سؤال مقارنة بخيارات، أو جدول، أو تمرين تحليل «في الحالة X استخدم Y لأن Z».

### اكتشف الخطأ (كود / منطق / حساب) — ضمن debug / exercise (≥5، hard)
**القاعدة/الاشتقاق (يحتوي خطأ):** … **اكتشف الخطأ:** … **التصحيح:** … + شرح الحل (3 نقاط).
أنواع الأخطاء: `logic`، `misconception`، `ambiguous_grammar`، `left_recursion`، `wrong_derivation`، `wrong_formula`.

### نظري منظم (تعريف + مكونات + مثال) — ضمن theory / mcq (≥10، medium)
### السؤال {N}: {السؤال}
**نموذج الإجابة:** 1. التعريف 2. المكونات/الشروط 3. مثال 4. متى نستخدم — هيكل ثابت يسهّل التصحيح الآلي.

## قواعد الكتل داخل الشرح

**📊 المخطط** — 3 أقسام بالترتيب: ما هذا المخطط؟ + جدول العُقد + جدول الروابط + بلوك `diagram`. types: `flowchart`، `decision-tree`. انظر SCHEMA.md v1.0 §Diagram.

**⚙️ الخطوات / الخوارزمية** — أسطر داخل fence ` ```algorithm ` بصيغة `1 | الخطوة | الأداة | ماذا يحدث` (سطر لكل خطوة). لا ترسم مربعات ولا أسهم — البارسر يحوّلها. انظر SCHEMA.md v1.0 §Algorithm.

**pseudo-code** — داخل code fence بـ `language_tag=text` (لخوارزميات التحليل).

**الفهم الخاطئ ❌** / **الفهم الصحيح ✅** — سطر واحد لكل منهما.

#### مهم للامتحان ⚠️ / #### نقطة مهمة ⚠️ / #### ملاحظة: / #### الدرس المستفاد:
← `blockquote` يتبع كل منها مباشرةً.

**🤔 تفعيل الفهم** — استخدمه **≥3 مرات** في كل محاضرة.

كود ناقص — ضع `_______` أو `// (N)` مكان السطر الناقص.

تصحيح كود — قدّم القاعدة/الاشتقاق الخاطئ أولاً ثم المصحّح.

عند وجود صورة في المحاضرة: اذكر رقم الصفحة وصف ما فيها.

**Q{N}:** سؤال / **A:** إجابة.

**💡 التشبيه** — بعد كل مفهوم مجرّد: جملة واحدة من الحياة اليومية + «وجه الشبه: X = Y». استخدمه بكثرة. انظر SCHEMA.md v1.0 §Analogy.

**⚖️ المقايضة** — جدول: المزايا | العيوب | متى تختاره. (فعّال لـ `LL vs LR`، `Top-Down vs Bottom-Up`). انظر SCHEMA.md v1.0 §Trade-off.

**🔄 قبل / بعد** — حالة/كود قبل العملية وبعدها + جملة «ماذا تغيّر؟». انظر SCHEMA.md v1.0 §Before-After.

**🔍 تتبع التنفيذ** (inline) — المدخل + جدول الخطوات + النتيجة. انظر SCHEMA.md v1.0 §Trace.

**📐 المعادلة** — LaTeX داخل `$$` أو fence `math`. يتبعها **الشرح:** بمعنى كل رمز (`First`، `Follow`، `E → E + T`، `S → aSBC`، `LR(0)`، `FOLLOW(F)`). انظر SCHEMA.md v1.0 §Equation.
