### 1. لماذا نحتاج طريقة أسرع لحساب الانتروبيا؟

#### النص الأصلي يقول:
> "The detailed calculations needed to choose an attribute to split on at a node in the evolving decision tree were illustrated in Section 4.3.3... For practical use a more efficient method is available which requires only a single table to be constructed for each categorical attribute at each node."

#### الشرح المبسّط:
في الفصل السابق، لحساب أي خاصية (`attribute`) نقسّم عليها العقدة، كنا نبني جدول بيانات منفصل لكل قيمة ممكنة من قيم تلك الخاصية، ثم نحسب entropy كل جدول فرعي على حدة. هذه الطريقة صحيحة لكنها بطيئة حسابياً. الطريقة الجديدة (`frequency table`) تبني **جدولاً واحداً فقط** لكل خاصية في كل عقدة، وتصل لنفس النتيجة الرياضية بحساب أبسط وأسرع.

**لماذا؟** لأن بناء جداول بيانات فرعية متعددة يكرر عملية المسح على البيانات أكثر من مرة؛ بينما جدول التكرار يمر على البيانات مرة واحدة فقط لعدّ التكرارات.

#### 💡 التشبيه:
> تخيل أنك تريد معرفة عدد الطلاب الناجحين والراسبين في كل شعبة من 3 شعب. بدلاً من عمل 3 كشوف حضور منفصلة (شعبة أ، ب، ج) ثم عدّ كل واحدة على حدة، تصنع **جدولاً واحداً** بأعمدة = الشعب وصفوف = النتيجة (ناجح/راسب)، وتعبّئه أثناء مرور واحد على قائمة الطلاب.
> **وجه الشبه:** كشف الحضور الواحد = جدول التكرار، كل شعبة = قيمة من قيم الخاصية.

### 2. بناء جدول التكرار (Frequency Table)

#### النص الأصلي يقول:
> "The cells of this table show the number of occurrences of each combination of class and attribute value in the training set."

#### الشرح المبسّط:
جدول التكرار لخاصية معينة (مثلاً `age` في بيانات `lens24`) يوضع فيه:
- **الأعمدة:** كل قيمة ممكنة للخاصية (`age = 1`, `age = 2`, `age = 3`).
- **الصفوف:** كل فئة تصنيف ممكنة (`Class 1`, `Class 2`, `Class 3`).
- **الخلايا:** عدد الحالات (instances) التي تحقق تلك القيمة وتلك الفئة معاً.
- **صف إضافي (Column sum):** مجموع كل عمود، أي إجمالي عدد الحالات لتلك القيمة من الخاصية بغض النظر عن الفئة.

| age = 1 | age = 2 | age = 3 |
| --- | --- | --- |
| Class 1: 2 | Class 1: 1 | Class 1: 1 |
| Class 2: 2 | Class 2: 2 | Class 2: 1 |
| Class 3: 4 | Class 3: 5 | Class 3: 6 |
| **Column sum: 8** | **Column sum: 8** | **Column sum: 8** |

**لماذا هذا التبسيط مهم؟** لأن كل ما نحتاجه لحساب `Enew` (متوسط الانتروبيا بعد التقسيم) موجود بالفعل داخل هذا الجدول الواحد، دون الحاجة لفصل البيانات فعلياً إلى مجموعات فرعية.

### 3. صيغة حساب `Enew` من جدول التكرار

#### النص الأصلي يقول:
> "(a) For every non-zero value V in the main body of the table... subtract V × log2 V. (b) For every non-zero value S in the column sum row, add S × log2 S. Finally, divide the total by N."

#### الشرح المبسّط:
الخطوات الثلاث لحساب `Enew`:
1. لكل قيمة غير صفرية V داخل جسم الجدول (خانات الفئات، وليس صف المجموع): اطرح `V·log2(V)`.
2. لكل مجموع عمود غير صفري S: أضف `S·log2(S)`.
3. اقسم الناتج الكلي على N (إجمالي عدد الحالات في مجموعة التدريب).

**لماذا نستثني القيم الصفرية؟** لأن `log2(0)` غير معرّف رياضياً، والفئة الفارغة أصلاً لا تساهم بأي معلومة إحصائية، فمن المنطقي تجاهلها بدل أن تسبب خطأ حسابي.

#### 📐 المعادلة: صيغة `Enew` بجدول التكرار

$$
E_{new} = \frac{-\sum_{j=1}^{V}\sum_{i=1}^{K} f_{ij}\log_2 f_{ij} \; + \; \sum_{j=1}^{V} N_j \log_2 N_j}{N}
$$

**الشرح:**
> `V` = عدد قيم الخاصية | `K` = عدد الفئات (classes) | `f_ij` = عدد الحالات للفئة `i` عند القيمة `j` (خلية في جسم الجدول) | `N_j` = مجموع العمود `j` (Column sum) | `N` = إجمالي عدد الحالات في مجموعة التدريب.

#### مثال تطبيقي (خاصية `age`، بيانات `lens24`):

```algorithm
1 | بناء جدول التكرار لـ age | frequency table | يُحسب عدد كل (class, age value) وصف مجموع الأعمدة
2 | طرح V.log2(V) لكل خلية غير صفرية | حساب يدوي/كود | -2log2(2) -1log2(1) -1log2(1) -2log2(2) -2log2(2) -1log2(1) -4log2(4) -5log2(5) -6log2(6)
3 | إضافة S.log2(S) لكل مجموع عمود | حساب يدوي/كود | +8log2(8) +8log2(8) +8log2(8)
4 | قسمة الناتج على N=24 | حساب يدوي/كود | الناتج النهائي Enew = 1.2867 bits
```

#### نقاط التنفيذ:
- النتيجة (1.2867 bits) مطابقة تماماً لما كانت ستُحسب بالطريقة القديمة (جداول فرعية منفصلة لكل قيمة) — وهذا هو **إثبات التكافؤ** الذي سنراه في القسم التالي.
- هذه القيمة تمثل "الفوضى" (impurity) المتوقعة بعد التقسيم على `age`؛ كلما كانت أصغر، كان التقسيم أفضل.

### 4. إثبات التكافؤ بين الطريقتين (Proof of Equivalence)

#### النص الأصلي يقول:
> "It remains to be proved that this method always gives the same value of Enew as the basic method described in Chapter 4."

#### الشرح المبسّط:
الإثبات يعتمد على إعادة كتابة الصيغة الأساسية للانتروبيا (وزن كل مجموعة فرعية × إنتروبيا تلك المجموعة) خطوة بخطوة، وإظهار أنها **جبرياً** تساوي صيغة جدول التكرار.

خطوات الإثبات المختصرة:
1. `Ej` (إنتروبيا المجموعة الفرعية `j`) = `-Σ (f_ij/N_j)·log2(f_ij/N_j)`.
2. `Enew` = المتوسط الموزون = `Σ (N_j/N)·Ej`.
3. بالتعويض والتبسيط: `Enew = -Σ Σ (f_ij/N)·log2(f_ij) + Σ (N_j/N)·log2(N_j)`.
4. هذا يطابق تماماً صيغة جدول التكرار المذكورة في القسم 3.

**لماذا هذا الإثبات مهم؟** لأنه يمنحنا الثقة الرياضية بأن التبسيط الحسابي (جدول واحد بدل عدة جداول) **لا يفقدنا أي دقة**؛ فهو اختصار حسابي بحت وليس تقريباً.

#### مهم للامتحان ⚠️:
> غالباً ما يُطلب في الامتحان إعادة اشتقاق هذه الخطوات جبرياً أو تطبيقها على مثال رقمي جديد — احفظ التسلسل: `Ej` → `Enew` كمتوسط موزون → التعويض → التبسيط.

### 5. ملاحظة حول الأصفار (A Note on Zeros)

#### النص الأصلي يقول:
> "If a complete column of the frequency table is zero it means that the categorical attribute never takes one of its possible values at the node under consideration. Any such columns are ignored."

#### الشرح المبسّط:
حالتان يجب التمييز بينهما:
- **خلية واحدة = صفر:** فئة معينة غير موجودة عند تلك القيمة من الخاصية → تُستبعد فقط تلك الخلية من الجمع (لأن `0·log2(0)` تُعامل كـ 0).
- **عمود كامل = صفر:** الخاصية لم تأخذ تلك القيمة إطلاقاً في هذه العقدة (المجموعة الفرعية فارغة تماماً) → يُتجاهل العمود بالكامل، وهذا يقابل تجاهل المجموعات الفرعية الفارغة أثناء بناء الشجرة.

**الفهم الخاطئ الشائع ❌:** الاعتقاد أن وجود صفر في الجدول يعني وجود خطأ في البيانات أو يجب استبدال القيمة بشيء آخر.
**الفهم الصحيح ✅:** الصفر معلومة صحيحة وطبيعية (غياب تلك الفئة/القيمة)، ويُعامل بقاعدة رياضية واضحة (تجاهل، وليس تعويض).

### 6. معيار جيني (Gini Index of Diversity)

#### النص الأصلي يقول:
> "If there are K classes, with the probability of the ith class being p_i, the Gini Index is defined as 1 − Σ p_i². This is a measure of the 'impurity' of a dataset."

#### الشرح المبسّط:
`Gini Index` مقياس بديل لـ `entropy` لقياس "عدم النقاء" (impurity) في مجموعة بيانات:
- أصغر قيمة = **صفر**، عندما تكون كل الحالات من فئة واحدة فقط (نقاء تام).
- أكبر قيمة = `1 − 1/K`، عندما تكون الفئات موزعة بالتساوي (أقصى فوضى ممكنة).

**لماذا نحتاج بديلاً عن entropy؟** لأن حساب `Gini` لا يتطلب دالة لوغاريتمية (`log2`)، فهو أسرع حسابياً قليلاً، وله خصائص إحصائية مختلفة عن entropy قد تناسب بعض مسائل التصنيف بشكل أفضل.

#### 💡 التشبيه:
> تخيل كيساً فيه كرات ملونة. إذا كانت كل الكرات بنفس اللون، فاحتمال سحب كرتين بلونين مختلفين = صفر (Gini = 0، نقاء تام). كلما زاد تنوع الألوان بالتساوي، زاد احتمال سحب لونين مختلفين (Gini يرتفع نحو أقصاه).
> **وجه الشبه:** تنوع ألوان الكرات = تنوع الفئات (classes)، احتمال سحب لونين مختلفين = قيمة Gini.

#### 📐 المعادلة: `Gini Index`

$$
Gini = 1 - \sum_{i=1}^{K} p_i^2
$$

**الشرح:**
> `K` = عدد الفئات | `p_i` = احتمال (نسبة) الفئة `i` في مجموعة البيانات.

### 7. حساب `Gininew` باستخدام جدول التكرار

#### النص الأصلي يقول:
> "The weighted average value of the Gini Index for the subsets resulting from splitting on the attribute is Gininew = ... = 1 − (1/N) Σ(1/Nj) Σ f_ij²"

#### الشرح المبسّط:
نفس جدول التكرار المستخدم لـ entropy يُستخدم لحساب `Gininew`، بخطوات مختلفة قليلاً:
1. لكل عمود غير فارغ: اجمع مربعات القيم داخل جسم الجدول، ثم اقسم الناتج على مجموع ذلك العمود.
2. اجمع النواتج من كل الأعمدة، ثم اقسم على N.
3. اطرح الناتج من 1 → هذه هي قيمة `Gininew`.

#### 📐 المعادلة: `Gininew` بجدول التكرار

$$
Gini_{new} = 1 - \frac{1}{N}\sum_{j=1}^{V}\frac{1}{N_j}\sum_{i=1}^{K} f_{ij}^{2}
$$

**الشرح:**
> نفس رموز جدول entropy: `f_ij` من جسم الجدول، `N_j` مجموع العمود، `N` الإجمالي.

```algorithm
1 | لكل عمود: اجمع مربعات القيم داخله | حساب يدوي/كود | age=1: 2²+2²+4²=24 | age=2: 1²+2²+5²=30 | age=3: 1²+1²+6²=38
2 | قسمة كل مجموع على مجموع العمود N_j | حساب يدوي/كود | 24/8=3 | 30/8=3.75 | 38/8=4.75
3 | جمع النواتج وقسمتها على N | حساب يدوي/كود | (3+3.75+4.75)/24 = 0.4792
4 | الطرح من 1 | حساب يدوي/كود | Gininew = 1 - 0.4792 = 0.5208
```

#### نقاط التنفيذ:
- الخاصية المختارة هي التي تعطي **أكبر انخفاض** في قيمة Gini، أي `Ginistart − Gininew` الأكبر — وليس أصغر `Gininew` بمفرده (رغم أنه في حال ثبات `Ginistart` يعطيان نفس الترتيب).
- لبيانات `lens24`: التخفيضات كانت `age=0.0174`, `specRx=0.0104`, `astig=0.0729`, `tears=0.2118` → الخاصية المختارة `tears` (نفس اختيار entropy في هذا المثال).

#### ⚖️ المقايضة: Entropy مقابل Gini Index

| | Entropy | Gini Index |
| --- | --- | --- |
| المزايا | مبني على نظرية المعلومات، تفسير احتمالي واضح (bits) | لا يحتاج `log`، حساب أسرع قليلاً |
| العيوب | يحتاج دالة لوغاريتمية (أبطأ حسابياً)، متحيز نحو الخصائص كثيرة القيم | أقل ارتباطاً بنظرية المعلومات، قد يعطي نتائج مختلفة قليلاً عن entropy في بعض الحالات |
| متى تختاره | عند الحاجة لتفسير قائم على "كمية المعلومة" (bits) أو عند استخدام C4.5 | عند أولوية السرعة الحسابية (مثل CART) |

### 8. التحيز الاستقرائي (Inductive Bias)

#### النص الأصلي يقول:
> "In practice we have a strong preference for hypothesising certain kinds of solution rather than others... it is an inductive bias, i.e. a preference for one choice rather than another, which is not determined by the data itself... There is no neutral, unbiased method."

#### الشرح المبسّط:
`Inductive Bias` هو أي **افتراض أو تفضيل مسبق** يفرضه أسلوب أو خوارزمية معينة عند التعميم من بيانات محدودة إلى قاعدة عامة، وهذا التفضيل **لا يأتي من البيانات نفسها** بل من طبيعة الطريقة المستخدمة.

المثال الكلاسيكي في المحاضرة: متتالية `1, 4, 9, 16, ...` — معظم الناس يجيبون `25` (المربعات الكاملة) رغم أن الجواب "الصحيح" رياضياً بحسب صيغة كثيرة الحدود المعطاة هو `20`. هذا يوضح أننا نحمل تحيزاً فطرياً نحو الأنماط "البسيطة والمألوفة".

**لماذا هذا مهم لأشجار القرار؟** لأن كل معيار اختيار خاصية (entropy، Gini، إلخ) هو شكل من أشكال `inductive bias`. تحديداً: **entropy متحيز نحو اختيار الخصائص التي لديها عدد كبير من القيم الممكنة**، حتى لو كانت تلك الخصائص غير مفيدة عملياً (مثال: خاصية "مكان الميلاد" في بيانات طبية).

#### 💡 التشبيه:
> اختبار الذكاء المدرسي "أكمل المتتالية" يفترض ضمنياً أنك ستفضل الأنماط الرياضية "الجميلة" (مربعات، مكعبات) على أي حل رياضي آخر صحيح تقنياً. هذا تماماً كيف يفترض `entropy` ضمنياً تفضيل الخصائص متعددة القيم دون أن يكون ذلك مبرَّراً دائماً.
> **وجه الشبه:** تفضيل "25" في الاختبار = تفضيل entropy للخصائص كثيرة القيم، كلاهما تحيز غير مبرر بالكامل من المعطيات وحدها.

#### نقطة مهمة ⚠️:
> لا توجد طريقة اختيار خاصية "محايدة تماماً" أو خالية من التحيز — كل طريقة تحمل تحيزها الخاص، والمطلوب هو **معرفة طبيعة هذا التحيز** واختيار الطريقة المناسبة لكل مسألة.

### 9. نسبة الكسب (Gain Ratio) و`Split Information`

#### النص الأصلي يقول:
> "In order to reduce the effect of the bias resulting from the use of information gain, a variant known as Gain Ratio was introduced by... Ross Quinlan in his influential system C4.5. Gain Ratio adjusts the information gain for each attribute to allow for the breadth and uniformity of the attribute values."

#### الشرح المبسّط:
`Gain Ratio` = طريقة لتصحيح تحيز `Information Gain` (= `Estart − Enew`) نحو الخصائص كثيرة القيم، عبر قسمته على `Split Information`:

$$
\text{Gain Ratio} = \frac{\text{Information Gain}}{\text{Split Information}}
$$

**الشرح:**
> `Information Gain` = مقدار انخفاض الانتروبيا بعد التقسيم (`Estart − Enew`) | `Split Information` = مقياس "تشتت" قيم الخاصية نفسها (بغض النظر عن الفئات).

`Split Information` تُحسب من صف "مجموع الأعمدة" فقط في جدول التكرار:

$$
\text{Split Information} = -\sum_{j=1}^{V} \frac{N_j}{N}\log_2\left(\frac{N_j}{N}\right)
$$

#### مثال تطبيقي (خاصية `age`):

```algorithm
1 | حساب Information Gain | Estart - Enew | 1.3261 - 1.2867 = 0.0394
2 | حساب Split Information من صف مجموع الأعمدة | -(8/24)log2(8/24) × 3 | = 1.5850
3 | قسمة Information Gain على Split Information | Gain Ratio | 0.0394 / 1.5850 = 0.0249
```

**لماذا يصحح هذا التحيز؟** لأن `Split Information` تزداد كلما زاد عدد قيم الخاصية وكانت موزعة بالتساوي (أقصى قيمة لها = `log2(M)` عند `M` قيمة متساوية التكرار)؛ وبما أن `Information Gain` يميل هو أيضاً للارتفاع مع كثرة القيم، فإن القسمة "توازن" هذا الارتفاع المصطنع.

#### خصائص `Split Information` (ملخص):
| الحالة | قيمة Split Information |
| --- | --- |
| قيمة واحدة فقط للخاصية | صفر |
| توزيع غير متساوٍ لعدد ثابت من القيم | قيمة متوسطة (تزداد كلما اقترب التوزيع من التساوي) |
| توزيع متساوٍ تماماً على `M` قيمة | `log2(M)` (أقصى قيمة ممكنة) |

#### 🤔 تفعيل الفهم (اسأل نفسك):
> **سؤال:** لماذا لا تكون `Split Information` أبداً سالبة؟
> **لماذا هذا مهم؟** لأنها مجموع حدود من شكل `-(x)log2(x)` حيث `0<x≤1`، وهذا الشكل دائماً `≥ 0` (لأن `log2(x) ≤ 0` عندما `x ≤ 1`، فيصبح الحد الكلي موجباً أو صفراً)، مما يضمن أن `Gain Ratio` معرّف رياضياً بشكل سليم طالما `Split Information ≠ 0`.

#### نقطة مهمة ⚠️:
> عندما تكون قيمة `Split Information` قريبة من الصفر (خاصية بقيمة واحدة فعلياً)، يصبح `Gain Ratio` غير مستقر (قسمة على رقم قريب من الصفر) — بعض التطبيقات العملية لـ C4.5 تضع حداً أدنى لتفادي هذه المشكلة.

---

### 10. عدد القواعد الناتجة بمعايير الاختيار المختلفة

#### النص الأصلي يقول:
> "For many datasets Information Gain (i.e. entropy reduction) and Gain Ratio give the same results. For others using Gain Ratio can give a significantly smaller decision tree. However... neither Information Gain nor Gain Ratio invariably gives the smallest decision tree."

#### الشرح المبسّط:
مقارنة عملية على مجموعات بيانات حقيقية (`contact lenses`, `lens24`, `chess`, `vote`, `monk1/2/3`) أظهرت:
- أحياناً `Gain Ratio` يعطي شجرة أصغر (عدد قواعد أقل) من `Entropy`، وأحياناً العكس.
- لا توجد طريقة تفوز دائماً — وهذا امتداد مباشر لفكرة `Inductive Bias`: كل معيار مناسب لبعض المسائل وغير مناسب لأخرى.

**لماذا هذا مهم عملياً؟** لأنه يعلمنا ألا "نثق" بمعيار واحد بشكل أعمى في كل المشاريع، بل نختبر أكثر من معيار (`criterion` في `sklearn`) عند بناء نموذج فعلي.

### 11. الفروع المفقودة (Missing Branches)

#### النص الأصلي يقول:
> "It may be that for the instances being considered there... there are no cases where attribute Z has the value d. In that case only three branches would be generated... This corresponds to an empty subset of instances."

#### الشرح المبسّط:
عند التقسيم على خاصية لها `V` قيم ممكنة، لكن إحدى القيم **لا تظهر إطلاقاً** في الحالات الوارِدة عند تلك العقدة تحديداً (وليس في كامل البيانات الأصلية)، فإن الفرع المقابل لتلك القيمة **لا يُنشأ إطلاقاً**. هذا مرتبط مباشرة بقاعدة "تجاهل الأعمدة الصفرية بالكامل" في جدول التكرار (القسم 5).

**النتيجة العملية:** أي بيانة (`instance`) جديدة غير مرئية (`unseen`) تحمل تلك القيمة المفقودة عند تلك النقطة من الشجرة **لن يكون لها أي فرع تسلكه**، فتبقى **غير مصنَّفة** (`unclassified`).

#### 🔄 قبل / بعد: التقسيم على خاصية Z ذات 4 قيم عند وجود فرع فارغ
**قبل:**
```r
# Z has possible values: a, b, c, d
# expected branches = 4 (a, b, c, d)
```

**بعد:**
```r
# instances at this node never have Z = d
# actual branches created = 3 (a, b, c only)
```

**ماذا تغيّر؟** عدد الفروع الفعلية أصبح أقل من عدد القيم النظرية للخاصية، لأن الفرع `d` مطابق لمجموعة فرعية فارغة تماماً.

#### الدرس المستفاد:
> الحل العملي المقترح في المحاضرة: بدل ترك الحالة غير المصنّفة كما هي، يمكن لنظام استقراء عملي أن يعطيها **تصنيفاً افتراضياً** (default classification) يساوي أكبر فئة (majority class) في العقدة الأقرب لها.

### أبرز النقاط الذهبية
1. جدول التكرار يعطي نفس نتيجة الطريقة الأساسية (مُثبت جبرياً) لكن بحساب أسرع بمرور واحد على البيانات.
2. الأعمدة الصفرية بالكامل في جدول التكرار = فروع مفقودة فعلياً في الشجرة.
3. لا توجد طريقة اختيار خاصية محايدة تماماً — كل طريقة تحمل `inductive bias` خاصاً بها.
4. `Entropy` متحيز نحو الخصائص كثيرة القيم؛ `Gain Ratio` يحاول تصحيح هذا لكن دون ضمان مطلق للأفضلية.
5. اختيار الخاصية دائماً يعتمد على **أكبر انخفاض** (سواء في entropy أو Gini)، وليس أصغر قيمة مطلقة بعد التقسيم فقط.

### الأخطاء الشائعة عند الطلاب ⚠️
| الخطأ | التصحيح |
| --- | --- |
| اعتقاد أن جدول التكرار "تقريب" لطريقة الفصل 4 | هو تكافؤ رياضي مُثبت، ليس تقريباً |
| نسيان تجاهل الأعمدة الصفرية بالكامل عند حساب Split Information | يجب استبعادها تماماً من الجمع، تماماً كما في حساب Enew |
| الخلط بين `Gininew` الأصغر و`أكبر انخفاض` في Gini | القرار يعتمد على `Ginistart − Gininew` الأكبر، خصوصاً عند مقارنة عقد مختلفة |
| اعتبار Gain Ratio دائماً أفضل من Information Gain | الجدول 5.5 يثبت أن لا طريقة تفوز في كل الحالات |

### خطوات وإجراءات المحاضرة

#### ⚙️ الخطوات / الخوارزمية: حساب Enew بطريقة جدول التكرار
> الهدف: حساب متوسط الانتروبيا بعد التقسيم دون بناء جداول فرعية متعددة.
```algorithm
1 | بناء جدول تكرار واحد للخاصية | frequency table | عدّ (class, attribute value) لكل حالة في مجموعة التدريب
2 | جمع حدود جسم الجدول | -V log2 V | لكل خلية غير صفرية داخل جسم الجدول (وليس صف المجموع)
3 | جمع حدود صف المجموع | +S log2 S | لكل مجموع عمود غير صفري
4 | القسمة على N | Enew = المجموع / N | N = إجمالي عدد الحالات في مجموعة التدريب
```
#### نقاط التنفيذ:
- تجاهل أي خلية أو عمود = صفر تماماً من الجمع (لا تُدرَج كـ 0 في اللوغاريتم).
- النتيجة يجب مقارنتها بـ `Estart` (انتروبيا العقدة قبل التقسيم) للحصول على `Information Gain`.

#### ⚙️ الخطوات / الخوارزمية: حساب Gininew بطريقة جدول التكرار
> الهدف: حساب متوسط عدم النقاء (Gini) بعد التقسيم باستخدام نفس جدول التكرار.
```algorithm
1 | لكل عمود غير فارغ: مجموع مربعات خلايا العمود | Σ f_ij² | يُحسب لكل قيمة من قيم الخاصية على حدة
2 | قسمة كل مجموع على مجموع عموده | (Σ f_ij²)/N_j | يمثل مساهمة ذلك العمود قبل الجمع الكلي
3 | جمع النواتج من كل الأعمدة وقسمتها على N | (1/N) Σ (Σf_ij²/N_j) | مجموع مرجّح على كل الأعمدة
4 | طرح الناتج من 1 | Gininew = 1 - الناتج | القيمة النهائية لعدم النقاء بعد التقسيم
```
#### نقاط التنفيذ:
- الخاصية الأفضل هي صاحبة أكبر فرق `Ginistart − Gininew`، وليس أصغر `Gininew` وحده.

#### ⚙️ الخطوات / الخوارزمية: حساب Gain Ratio
> الهدف: تصحيح تحيز Information Gain نحو الخصائص كثيرة القيم.
```algorithm
1 | حساب Information Gain | Estart - Enew | يُحسب لكل خاصية مرشحة للتقسيم
2 | حساب Split Information من صف مجموع الأعمدة | -Σ (Nj/N) log2(Nj/N) | يعتمد فقط على توزيع قيم الخاصية، بغض النظر عن الفئات
3 | القسمة | Gain Ratio = Information Gain / Split Information | يُختار أكبر Gain Ratio بين كل الخصائص المرشحة
```
#### نقاط التنفيذ:
- إذا كانت Split Information قريبة من الصفر (خاصية شبه ثابتة القيمة)، يصبح الناتج غير مستقر عددياً.

### أنماط الأكواد والبنى المتكررة
| النمط | البنية الأساسية | متى تستخدمه |
| --- | --- | --- |
| بناء جدول تكرار (`pandas.crosstab` / `table()` في R) | `pd.crosstab(df['class'], df[attribute])` | عند حساب Enew أو Gininew يدوياً قبل استخدام مكتبة جاهزة |
| اختيار `criterion` في شجرة قرار جاهزة | `DecisionTreeClassifier(criterion='entropy'|'gini')` | عند بناء نموذج فعلي بدل الحساب اليدوي |
| حساب `log2` بأمان لتفادي قيم صفرية | `np.where(x > 0, x*np.log2(x), 0)` | داخل أي دالة entropy أو split information مخصصة |

### أنماط التعامل والسلوك
| السيناريو | التعامل الصحيح | لماذا؟ |
| --- | --- | --- |
| عمود كامل في جدول التكرار = صفر | تجاهل العمود بالكامل من كل الحسابات (Enew, Gininew, Split Information) | يمثل فرعاً غير موجود أصلاً (subset فارغ) |
| حالة اختبار جديدة تسلك فرعاً غير موجود في الشجرة | تصنيفها كـ "غير مصنّفة" أو إعطاؤها التصنيف الافتراضي (أكبر فئة) | هذا أفضل من تصنيفها بشكل خاطئ قسراً |
| خاصية بعدد كبير من القيم الفريدة (مثل معرّف فريد لكل حالة) | الحذر من الاعتماد فقط على Information Gain، واستخدام Gain Ratio كتحقق إضافي | Information Gain متحيز بشدة نحو هذه الخصائص |

### الأفكار الرئيسية الشاملة
لا يوجد معيار اختيار خاصية "صحيح مطلقاً"؛ الاختيار بين `Entropy`, `Gini`, أو `Gain Ratio` هو اختيار هندسي عملي يعتمد على طبيعة البيانات، وينبغي دوماً مراعاة أن أي معيار يحمل تحيزاً استقرائياً (Inductive Bias) خاصاً به يجب فهمه قبل الاعتماد عليه في مشروع حقيقي.

## الجزء الثالث: أسئلة اختيار من متعدد (MCQ)

> **20 سؤالاً** — مستوى: medium/hard. التوزيع: مقارنات 20% | سيناريو كود 25% | تطبيق 35% | تتبع خوارزميات 20%.

### السؤال 1 (medium) — تطبيق
ما هو الغرض الأساسي من استخدام `frequency table` بدلاً من الطريقة الأساسية في الفصل 4؟
أ) الحصول على نتيجة أكثر دقة  ب) تقليل عدد الحسابات مع الحفاظ على نفس النتيجة  ج) تجنب استخدام `log2`  د) تحويل البيانات إلى فئات جديدة
**الإجابة الصحيحة: ب**
**التعليل:** جدول التكرار مُثبت أنه يعطي نفس القيمة تماماً (وليس دقة أعلى، أ خاطئ)، ولا يزال يستخدم `log2` (ج خاطئ)، ولا يُنشئ فئات جديدة (د خاطئ). هدفه فقط تقليل الحساب.

### السؤال 2 (medium) — تطبيق
في جدول التكرار، ماذا يمثل "Column sum"؟
أ) عدد الفئات الكلي  ب) N (إجمالي كل البيانات)  ج) N_j، عدد الحالات لتلك القيمة من الخاصية  د) قيمة log2 للعمود
**الإجابة الصحيحة: ج**
**التعليل:** Column sum هو مجموع خلايا عمود واحد فقط، أي N_j لتلك القيمة تحديداً، وليس N الإجمالي (ب) ولا عدد الفئات (أ) ولا لوغاريتم (د).

### السؤال 3 (hard) — تتبع خوارزميات
عند حساب Enew من جدول تكرار، ماذا يحدث لخلية قيمتها صفر؟
أ) تُحسب كـ 0×log2(0)=0 وتُضاف للجمع  ب) تُستبعد تماماً من الجمع لأن log2(0) غير معرّف  ج) تُستبدل بـ 1 قبل الحساب  د) تُسبب خطأ ويجب إيقاف الحساب
**الإجابة الصحيحة: ب**
**التعليل:** المحاضرة تنص صراحة أن القيم غير الصفرية فقط تُدرج في الجمع؛ الصفر يُستبعد وليس يُحسب كصفر رياضياً لأن log2(0) غير معرف أصلاً.

### السؤال 4 (medium) — مقارنة
ما الفرق الجوهري بين حساب Enew وحساب Gininew من نفس جدول التكرار؟
أ) Gininew لا يستخدم صف مجموع الأعمدة بنفس طريقة Enew  ب) Enew يحتاج log2 بينما Gininew يعتمد على مربعات القيم  ج) لا يوجد فرق، كلاهما نفس الصيغة  د) Gininew يتطلب جدول تكرار مختلف كلياً
**الإجابة الصحيحة: ب**
**التعليل:** Enew مبني على `V log2 V` و`S log2 S`؛ Gininew مبني على مجموع مربعات `f_ij²` مقسومة على `N_j`. لا حاجة للوغاريتم في Gini (أ وج ود غير دقيقة).

### السؤال 5 (medium) — تطبيق
لبيانات lens24 (N=24)، إذا كانت جميع الأعمدة الثلاثة لخاصية age تساوي 8 لكل منها، فإن قيمة Split Information لهذه الخاصية تساوي تقريباً؟
أ) 1.0  ب) 1.585  ج) 0  د) 3.0
**الإجابة الصحيحة: ب**
**التعليل:** كما ورد في المحاضرة تماماً: `-(8/24)log2(8/24)` × 3 = 1.5850. القيمة 1.0 (أ) تقابل توزيعاً على قيمتين متساويتين، و0 (ج) يقابل قيمة واحدة فقط، و3.0 (د) رقم عشوائي غير مطابق.

### السؤال 6 (hard) — سيناريو كود
```python
def entropy_term(v):
    return v * np.log2(v) if v > 0 else 0
```
ما الدور الوظيفي لهذا الشرط `if v > 0`؟
أ) لتسريع الكود فقط  ب) لتفادي القيم السالبة  ج) لتطبيق قاعدة استبعاد الأصفار من حساب الانتروبيا  د) لتحويل v إلى عدد صحيح
**الإجابة الصحيحة: ج**
**التعليل:** هذا الشرط يطبق تماماً "ملاحظة الأصفار" في المحاضرة: القيم الصفرية تُستبعد لأن log2(0) غير معرّف، وليس لأسباب تتعلق بالسرعة (أ) أو السالب (ب) أو التحويل لعدد صحيح (د).

### السؤال 7 (medium) — مقارنة
أي العبارات صحيحة بخصوص Gain Ratio مقارنة بـ Information Gain؟
أ) Gain Ratio دائماً أكبر من Information Gain  ب) Gain Ratio يقلل التحيز نحو الخصائص كثيرة القيم لكن لا يضمن أصغر شجرة دائماً  ج) Information Gain لا علاقة له بالتحيز الاستقرائي  د) Gain Ratio يُستخدم فقط مع Gini Index
**الإجابة الصحيحة: ب**
**التعليل:** هذا مباشرة من نتائج الجدول 5.5 في المحاضرة؛ Gain Ratio لا يفوز دائماً (أ خاطئة إحصائياً وحسابياً)، وInformation Gain نفسه مثال على تحيز استقرائي (ج خاطئة)، ولا علاقة له حصرياً بـ Gini (د خاطئة).

### السؤال 8 (hard) — تطبيق
لماذا يُعتبر اختيار "مكان الميلاد" كخاصية تقسيم أولى في شجرة قرار طبية مشكلة نموذجية لتحيز entropy؟
أ) لأن مكان الميلاد بيانات حساسة أخلاقياً فقط  ب) لأن الخاصية كثيرة القيم فتُعطي Information Gain مرتفعاً زائفاً رغم تأثيرها الفعلي الضعيف  ج) لأن entropy لا يمكنه التعامل مع البيانات النصية  د) لأن Gini Index يرفض هذه الخاصية تلقائياً

### سؤال تصحيح 6 — misconception

    return node.majority_class
```
**شرح الحل:**

### تمرين 2: code_fix — دالة entropy عامة
**السيناريو / المطلوب:**
لديك دالة entropy عامة بها خطأ يمنعها من العمل مع أكثر من فئتين:

def entropy(class_counts, N):
    p1 = class_counts[0] / N
    p2 = class_counts[1] / N

def entropy(class_counts, N):
    total = 0
    for c in class_counts:
        if c > 0:
            p = c / N
            total -= p * np.log2(p)
    return total
```

### تمرين 6: code_fix — Split Information مع أعمدة صفرية
**السيناريو / المطلوب:**
دالة Split Information التالية تفشل عند وجود عمود صفري ضمن col_sums (نتيجة NaN):
```python
def split_information(col_sums, N):
    p = col_sums / N
    return -np.sum(p * np.log2(p))
```
أصلحها لتتجاهل الأعمدة الصفرية بشكل صحيح.

**نموذج الحل:**
```python
def split_information(col_sums, N):
    total = 0
    for s in col_sums:
        if s > 0:
            p = s / N
            total -= p * np.log2(p)
    return total
```

## الجزء الخامس: أسئلة نظرية متوقعة بالامتحان

### السؤال 1: ما الفرق بين طريقة حساب Enew في الفصل 4 وطريقة جدول التكرار في هذا الفصل؟
**نموذج الإجابة:** 1. التعريف: كلاهما يحسب نفس القيمة (متوسط الانتروبيا بعد التقسيم)، لكن بطريقتين حسابيتين مختلفتين. 2. المكونات: الفصل 4 يبني جدولاً منفصلاً لكل قيمة من قيم الخاصية؛ جدول التكرار يبني جدولاً واحداً فقط. 3. مثال: بيانات lens24، خاصية age، كلا الطريقتين تعطيان Enew=1.2867. 4. متى نستخدم: نستخدم جدول التكرار دائماً عملياً لكفاءته الحسابية الأعلى، مع العلم أن التكافؤ مُثبت رياضياً.

### السؤال 2: عرّف Gini Index واذكر مداه.
**نموذج الإجابة:** 1. التعريف: `1 − Σp_i²` مقياس لعدم نقاء التصنيف. 2. المكونات/الشروط: يعتمد على احتمال كل فئة p_i. 3. مثال: توزيع متساوٍ بين 3 فئات → Gini = 1-1/3 = 0.667 (القيمة الكبرى). 4. متى نستخدم: عند بناء أشجار قرار بمعيار CART بدل entropy.

### السؤال 3: ما هو Inductive Bias ولماذا لا يمكن تجنبه؟
**نموذج الإجابة:** 1. التعريف: تفضيل لحل معين غير مشتق كلياً من البيانات، بل من طبيعة الطريقة. 2. المكونات: أي خوارزمية تعميم (مثل اختيار خاصية التقسيم) تحمل هذا التحيز حتماً. 3. مثال: تحيز entropy نحو الخصائص كثيرة القيم. 4. متى نستخدم: يجب فهمه دائماً قبل اختيار أي معيار احصائي، فلا توجد طريقة محايدة.

### السؤال 4: كيف تُحسب Split Information وما دورها؟
**نموذج الإجابة:** 1. التعريف: مقياس تشتت قيم الخاصية بغض النظر عن الفئات. 2. المكونات: `-Σ(N_j/N)log2(N_j/N)` من صف مجموع الأعمدة فقط. 3. مثال: قيمتان متساويتان → Split Information = 1.0. 4. متى نستخدم: كمقام في معادلة Gain Ratio لتصحيح تحيز Information Gain.

### السؤال 5: اشرح ظاهرة Missing Branches وتأثيرها على التصنيف.
**نموذج الإجابة:** 1. التعريف: عدم إنشاء فرع لقيمة خاصية غير موجودة فعلياً في الحالات الواصلة لعقدة معينة. 2. المكونات: مرتبطة بأعمدة صفرية بالكامل في جدول التكرار. 3. مثال: خاصية Z ذات 4 قيم، لكن فقط 3 تظهر فعلياً. 4. متى نستخدم: يجب معالجتها ببروتوكول تصنيف افتراضي (majority class) لتفادي حالات "غير مصنَّف".

### السؤال 6: ما الفرق بين Information Gain وGain Ratio من ناحية التحيز؟
**نموذج الإجابة:** 1. التعريف: Information Gain = Estart-Enew؛ Gain Ratio = Information Gain/Split Information. 2. المكونات: Gain Ratio يضيف تصحيحاً بقسمة على مقياس تشتت قيم الخاصية. 3. مثال: خاصية age تعطي Information Gain=0.0394 وGain Ratio=0.0249. 4. متى نستخدم: Gain Ratio عندما نخشى تحيزاً نحو خصائص كثيرة القيم (مثل C4.5).

### السؤال 7: لماذا لا تتضمن صيغة الانتروبيا القيم الصفرية؟
**نموذج الإجابة:** 1. التعريف: القيم الصفرية تعني غياب حالات لتلك الفئة/القيمة. 2. المكونات: `log2(0)` غير معرّف رياضياً. 3. مثال: خلية فارغة في جدول age لبيانات lens24. 4. متى نستخدم: نطبق قاعدة الاستبعاد في كل حسابات entropy وGini وSplit Information.

### السؤال 8: كيف يُثبت تكافؤ طريقة جدول التكرار مع الطريقة الأساسية؟
**نموذج الإجابة:** 1. التعريف: عبر إعادة كتابة صيغة `Enew` الموزونة جبرياً حتى تطابق صيغة جدول التكرار. 2. المكونات: يعتمد الإثبات على تفكيك `log2(f_ij/N_j)` إلى `log2(f_ij) - log2(N_j)`. 3. مثال: الاشتقاق الكامل في القسم 5.1.1 من الكتاب. 4. متى نستخدم: كأساس نظري يبرر استخدام الطريقة الأسرع دون قلق من فقدان الدقة.

### السؤال 9: ما الفائدة العملية لمعرفة أن لا معيار اختيار خاصية يتفوق دائماً؟
**نموذج الإجابة:** 1. التعريف: نتيجة تجريبية موثقة في جدول 5.5. 2. المكونات: مقارنة بين Entropy وGain Ratio عبر 7 مجموعات بيانات مختلفة. 3. مثال: مجموعة monk3 كانت أفضل مع Gain Ratio (25 قاعدة) مقابل Entropy (28 قاعدة). 4. متى نستخدم: يجب دائماً اختبار أكثر من معيار عملياً بدل الاعتماد الأعمى على معيار واحد.

### السؤال 10: ما العلاقة بين Ginistart وGininew في اختيار الخاصية؟
**نموذج الإجابة:** 1. التعريف: Ginistart قيمة العقدة قبل التقسيم؛ Gininew بعد التقسيم على خاصية معينة. 2. المكونات: القرار يعتمد على الفرق بينهما (التخفيض) وليس القيمة المطلقة لـ Gininew. 3. مثال: بيانات lens24، Gstart=0.5382، أفضل تخفيض لـtears=0.2118. 4. متى نستخدم: عند مقارنة خصائص متعددة لاختيار الأنسب للتقسيم عند عقدة معينة.

### السؤال 11: كيف تُفسَّر أقصى قيمة ممكنة لـSplit Information؟
**نموذج الإجابة:** 1. التعريف: تحدث عند توزيع متساوٍ تماماً لقيم الخاصية. 2. المكونات: تساوي `log2(M)` حيث M عدد القيم. 3. مثال: 4 قيم متساوية التكرار → Split Information = log2(4) = 2. 4. متى نستخدم: لفهم كيف تعاقب خاصية كثيرة القيم ومتوازنة التوزيع أكثر من خاصية قليلة القيم عند حساب Gain Ratio.

### السؤال 12: لماذا يمكن اعتبار مثال المتتالية الرياضية في المحاضرة مقدمة مناسبة لمفهوم Gain Ratio؟
**نموذج الإجابة:** 1. التعريف: يوضح فكرة التحيز غير المبرر بالكامل من المعطيات. 2. المكونات: يبني حدساً قبل تقديم مثال تقني (تحيز entropy). 3. مثال: تفضيل "25" رغم صحة "20" رياضياً. 4. متى نستخدم: كأداة تعليمية تمهيدية لفهم لماذا احتاج الباحثون (Quinlan) لتطوير Gain Ratio كتصحيح جزئي.

## الجزء الرابع: بطاقات سؤال وجواب (Q&A Cards)

**Q1:** ما الفرق بين مصطلحي mean وmedian في سياق عام؟
A: mean هو المتوسط الحسابي؛ median هو القيمة الوسطى بعد الترتيب.

**Q2:** ماذا تمثل Column sum في جدول التكرار؟
A: مجموع كل الحالات لقيمة معينة من الخاصية (N_j)، بغض النظر عن الفئة.

**Q3:** ما هي أصغر قيمة ممكنة لـGini Index؟
A: صفر، عند النقاء التام (فئة واحدة فقط).

**Q4:** ما هي أكبر قيمة ممكنة لـGini Index؟
A: `1 − 1/K`، عند توزيع متساوٍ للفئات.

**Q5:** ما هي صيغة Information Gain؟
A: `Estart − Enew`.

**Q6:** ما هي صيغة Gain Ratio؟
A: `Information Gain / Split Information`.

**Q7:** متى تكون Split Information تساوي صفراً؟
A: عندما تأخذ الخاصية قيمة واحدة فقط.

**Q8:** متى تكون Split Information عند أقصى قيمة لها؟
A: عند توزيع متساوٍ تماماً لجميع قيم الخاصية (تساوي log2(M)).

**Q9:** لماذا نستبعد القيم الصفرية عند حساب Enew؟
A: لأن `log2(0)` غير معرّف رياضياً، والقيمة الصفرية لا تحمل معلومة أصلاً.

**Q10:** من قدّم Gain Ratio ولماذا؟
A: Ross Quinlan في نظام C4.5، لتقليل تحيز Information Gain نحو الخصائص كثيرة القيم.

**Q11:** ماذا يعني وجود عمود كامل من الأصفار في جدول التكرار؟
A: أن الخاصية لم تأخذ تلك القيمة إطلاقاً في الحالات المعنيّة عند تلك العقدة (فرع مفقود).

**Q12:** ما هو Inductive Bias؟
A: تفضيل لحل معين لا يُشتق من البيانات وحدها، بل من طبيعة الطريقة المستخدمة.

**Q13:** ما البديل العملي عند وصول حالة اختبار لفرع مفقود في الشجرة؟
A: إعطاؤها تصنيفاً افتراضياً (majority class) بدل تركها غير مصنّفة.

**Q14:** ماذا يحدث لعدد فروع خاصية إذا كانت إحدى قيمها غير موجودة في الحالات الواصلة لعقدة معينة؟
A: لا يُنشأ فرع لتلك القيمة إطلاقاً؛ عدد الفروع الفعلي يقل عن عدد القيم النظري.

**Q15:** هل معيار Gain Ratio يضمن دائماً أصغر شجرة قرار ممكنة؟
A: لا، لا يوجد معيار (لا Entropy ولا Gain Ratio) يضمن ذلك في كل مجموعات البيانات.

**Q16:** ما الفرق الحسابي الأساسي بين Enew وGininew؟
A: Enew يستخدم `V log2 V`، بينما Gininew يستخدم مربعات القيم `f_ij²` دون لوغاريتم.

```r
# === script.R ===
# Reference implementation: frequency-table based Entropy, Gini, and Gain Ratio
# All comments in English as required

# --- Safe log2 helper: treats 0 * log2(0) as 0, matching "Note on Zeros" ---
safe_term <- function(x) {
  ifelse(x > 0, x * log2(x), 0)
}

# --- Enew from a frequency table ---
# freq_table: matrix, rows = classes, cols = attribute values
enew_from_table <- function(freq_table, N) {
  col_sums <- colSums(freq_table)
  body_sum <- sum(safe_term(freq_table))       # step (a): subtract V*log2(V)
  colsum_sum <- sum(safe_term(col_sums))       # step (b): add S*log2(S)
  (-body_sum + colsum_sum) / N
}

# --- Gini index for a single node given class probabilities ---
gini_index <- function(probs) {
  1 - sum(probs^2)
}

# --- Gininew from the same frequency table ---
gininew_from_table <- function(freq_table, N) {
  col_sums <- colSums(freq_table)
  contributions <- sapply(seq_along(col_sums), function(j) {
    sum(freq_table[, j]^2) / col_sums[j]
  })
  1 - sum(contributions) / N
}

# --- Split Information from column sums ---
split_information <- function(col_sums, N) {
  p <- col_sums / N
  -sum(safe_term(p))
}

# --- Gain Ratio ---
gain_ratio <- function(estart, enew, col_sums, N) {
  info_gain <- estart - enew
  si <- split_information(col_sums, N)
  if (si == 0) return(0)  # guard against division by zero
  info_gain / si
}

# --- Example: lens24 "age" attribute frequency table ---
age_table <- matrix(c(2, 2, 4,
                       1, 2, 5,
                       1, 1, 6),
                     nrow = 3, byrow = TRUE)
N <- 24
cat("Enew(age) =", enew_from_table(age_table, N), "\n")   # expect ~1.2867
cat("Gininew(age) =", gininew_from_table(age_table, N), "\n") # expect ~0.5208
```

```python
# === script.py ===
# Reference implementation: frequency-table based Entropy, Gini, and Gain Ratio
import numpy as np

def safe_term(x):
    """0 * log2(0) treated as 0, matching 'Note on Zeros'."""
    x = np.asarray(x, dtype=float)
    result = np.zeros_like(x)
    mask = x > 0
    result[mask] = x[mask] * np.log2(x[mask])
    return result

def enew_from_table(freq_table, N):
    """freq_table: 2D array, rows=classes, cols=attribute values."""
    freq_table = np.asarray(freq_table, dtype=float)
    col_sums = freq_table.sum(axis=0)
    body_sum = safe_term(freq_table).sum()      # step (a)
    colsum_sum = safe_term(col_sums).sum()      # step (b)
    return (-body_sum + colsum_sum) / N

def gini_index(probs):
    probs = np.asarray(probs, dtype=float)
    return 1 - np.sum(probs ** 2)

def gininew_from_table(freq_table, N):
    freq_table = np.asarray(freq_table, dtype=float)
    col_sums = freq_table.sum(axis=0)
    contributions = (freq_table ** 2).sum(axis=0) / col_sums
    return 1 - contributions.sum() / N

def split_information(col_sums, N):
    col_sums = np.asarray(col_sums, dtype=float)
    p = col_sums / N
    return -safe_term(p).sum()

def gain_ratio(estart, enew, col_sums, N):
    info_gain = estart - enew
    si = split_information(col_sums, N)
    if si == 0:
        return 0  # guard against division by zero
    return info_gain / si

if __name__ == "__main__":
    # Example: lens24 "age" attribute frequency table
    age_table = np.array([
        [2, 2, 4],
        [1, 2, 5],
        [1, 1, 6],
    ])
    N = 24
    print("Enew(age) =", enew_from_table(age_table, N))       # expect ~1.2867
    print("Gininew(age) =", gininew_from_table(age_table, N)) # expect ~0.5208
```
