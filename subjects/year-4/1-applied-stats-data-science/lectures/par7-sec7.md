### 1. المشكلة: لماذا نحتاج Discretisation؟

#### النص الأصلي يقول:
> "Many data mining algorithms, including the TDIDT tree generation algorithm require all attributes to take categorical values. However, in the real world many attributes are naturally continuous, e.g. height, weight, length, temperature and speed."

#### الشرح المبسّط:
خوارزمية `TDIDT` (وهي الخوارزمية الأساسية لبناء أشجار القرار في هذه المادة) تتعامل فقط مع متغيرات **فئوية** (`categorical`) مثل `sunny`/`rainy` أو `high`/`low`. لكن كثيرًا من البيانات الحقيقية أرقام مستمرة مثل درجة الحرارة أو الطول. إذاً نحتاج طريقة لتحويل الرقم المستمر إلى فئة.

**لماذا؟** لأن معالجة كل قيمة رقمية كفئة منفصلة (مثلاً `X = 7.2` كفئة قائمة بذاتها) يعني أن كل فئة سيكون لها عدد قليل جدًا من الأمثلة (ربما مثال واحد فقط)، وبالتالي القاعدة الناتجة `X = 7.2` لن تكون مفيدة للتنبؤ بأمثلة جديدة لم يسبق رؤيتها بنفس القيمة تمامًا.

#### 💡 التشبيه:
> تخيل أنك تصنّف أعمار الناس بدل تجميعهم في فئات مثل "طفل"، "شاب"، "بالغ" — تتعامل مع كل عمر بمفرده (5 سنوات، 5.1 سنة، 5.2 سنة...) فتحصل على آلاف الفئات الفارغة تقريبًا.
> **وجه الشبه:** قيمة `X = 7.2` المفردة = عمر "5.1 سنة" بالضبط — فئة لا تتكرر ولا تُفيد التعميم.

### 2. الحل القياسي: التقسيم إلى مجالات (Ranges)

#### النص الأصلي يقول:
> "The standard approach is to split the values of a continuous attribute into a number of non-overlapping ranges... X < 7, 7 ≤ X < 12, 12 ≤ X < 20 and X ≥ 20... the values 7, 12 and 20 are called cut values or cut points."

#### الشرح المبسّط:
بدل التعامل مع القيمة كرقم مفرد، نقسّم المجال الكامل للمتغير إلى عدد صغير من "الشرائح" غير المتداخلة. كل شريحة تصبح "فئة" (`categorical value`) واحدة. نقاط الفصل بين الشرائح (هنا 7، 12، 20) تُسمّى `cut points` أو `cut values`.

مثال آخر من النص: العمر → `infant`, `child`, `young`, `adult`, `middle-aged`, `old`. الطول → `very short`, `short`, `medium`, `tall`, `very tall`.

هذه العملية تُسمّى **`Discretisation`**.

**لماذا؟** لأنها تقلل عدد الفئات الممكنة إلى عدد معقول يجعل القواعد الناتجة (`rules`) قابلة للتعميم على بيانات جديدة، بدل أن تكون خاصة بقيمة واحدة رأيناها في بيانات التدريب فقط.

#### 🤔 تفعيل الفهم (اسأل نفسك):
> **سؤال:** لماذا لا نجعل عدد الفئات كبيرًا جدًا (مثلاً 100 فئة) لنحافظ على أكبر قدر من المعلومة؟
> **لماذا هذا مهم؟** لأن زيادة عدد الفئات تُعيدنا لنفس مشكلة القيم المفردة: فئات قليلة الأمثلة وقواعد لا تُعمَّم جيدًا (نفس فكرة الـ overfitting التي سنراها لاحقًا في التعلم الآلي).

### 4. طريقة المجالات متساوية العرض (Equal Width Intervals)

#### النص الأصلي يقول:
> "Suppose that we have a continuous attribute length, with values in the range from 0.3 to 6.6 inclusive... 0.3 ≤ length < 2.4 / 2.4 ≤ length < 4.5 / 4.5 ≤ length ≤ 6.6. This is known as the equal width intervals method."

#### الشرح المبسّط:
نأخذ المدى الكامل للمتغير (هنا من 0.3 إلى 6.6، أي مدى طوله 6.3) ونقسّمه إلى عدد ثابت من الأجزاء المتساوية في العرض (هنا 3 أجزاء، عرض كل جزء = 2.1 تقريبًا).

**لماذا فيها مشكلة؟** لأن اختيار عدد الأجزاء (3 هنا) اعتباطي — لماذا لا 2 أو 4 أو 12؟ والأخطر أنه إذا تجمّعت قيم كثيرة قريبة من نقطة الفصل (مثل قيم كثيرة بين 2.35 و2.45)، فإن قيمة `2.39999` ستُعامل بشكل مختلف تمامًا عن `2.40001` رغم أنه لا فرق حقيقي بينهما (خصوصًا إذا كان القياس غير دقيق أصلاً).

#### 💡 التشبيه:
> كأنك تقسّم صف طلاب حسب الطول إلى 3 مجموعات بعرض متساوٍ (كل 70 سم مجموعة) بغض النظر عن كيفية توزّع أطوال الطلاب فعليًا — قد تحصل على مجموعة فيها 90% من الطلاب ومجموعتين شبه فارغتين.
> **وجه الشبه:** تجاهل توزيع البيانات الفعلي = تجاهل أين "يتكدّس" الطلاب فعليًا حول أطوال معينة.

### 5. طريقة المجالات متساوية التكرار (Equal Frequency Intervals)

#### النص الأصلي يقول:
> "Another possibility would be to divide length into three ranges... so that there are the same number of instances in each of the three ranges... This is known as the equal frequency intervals method."

#### الشرح المبسّط:
بدل تقسيم *مدى القيم* بالتساوي، نقسّم *عدد الأمثلة* بالتساوي — أي كل شريحة تحتوي على نفس عدد الحالات تقريبًا (بغض النظر عن عرض الشريحة بالأرقام).

**لماذا أفضل قليلاً من السابقة؟** لأنها تراعي كثافة البيانات الفعلية (لا تترك شريحة شبه فارغة وأخرى مزدحمة). لكنها **لا تزال تعاني من نفس مشكلة الحساسية عند نقطة الفصل** — لماذا `2.99999` يُعامل مختلفًا عن `3.00001`؟

#### ⚖️ المقايضة: طرق التقسيم غير الخاضعة للتصنيف (Unsupervised Discretisation)

| | Equal Width Intervals | Equal Frequency Intervals |
| --- | --- | --- |
| المزايا | بسيطة، سهلة الفهم والتنفيذ | تراعي كثافة توزيع القيم الفعلي |
| العيوب | تتجاهل توزيع البيانات، قد تُنتج شرائح فارغة/مزدحمة | لا تزال حسّاسة عند نقاط الفصل، وعدد الشرائح اختياري |
| متى تختاره | عند عدم وجود معلومة عن الفئة (`class`) وتحتاج حلًا سريعًا وبسيطًا | عند رغبتك بتوازن عدد الأمثلة لكل فئة دون معلومات عن التصنيف |

#### مهم للامتحان ⚠️:
> **كلا الطريقتين "لا تستخدمان معلومة التصنيف (class)"** عند اختيار نقاط الفصل — وهذا هو ضعفهما الجوهري في مسائل التصنيف. الطرق الأفضل (`ChiMerge` والتقسيم المحلي في `TDIDT`) تستخدم التصنيف مباشرة.

### 6. مشكلة الحساسية الزائدة (Over-sensitivity) وفكرة "الفجوات" (Gaps)

#### النص الأصلي يقول:
> "The problem with any method of discretising continuous attributes is that of over-sensitivity... Ideally we would like to find 'gaps' in the range of values."

#### الشرح المبسّط:
المشكلة العامة: أي قيمة تقع مباشرة تحت نقطة الفصل ستُعامَل بشكل مختلف تمامًا عن قيمة تقع فوقها بقليل جدًا، دون سبب حقيقي. الحل المثالي (نظريًا) هو اختيار نقطة الفصل داخل "فجوة" في البيانات — منطقة لا توجد فيها أي قيم فعلية في بيانات التدريب، فتصبح نقطة الفصل بلا تأثير عملي حقيقي على أي حالة قريبة منها.

**المشكلة العملية:** الفجوة الموجودة في بيانات التدريب قد لا تتكرر في بيانات الاختبار (`unseen test data`). فإذا ظهرت قيم مثل 0.99 و1.05 و1.49 و1.51 في بيانات جديدة، فإن اختيار نقطة الفصل بين 1.0 أو 1.5 يصبح حاسمًا وقد يُغيّر النتيجة كليًا رغم أن الاختيار كان اعتباطيًا أصلاً.

#### الفهم الخاطئ الشائع ❌: إيجاد فجوة في بيانات التدريب يحل مشكلة الحساسية نهائيًا.
#### الفهم الصحيح ✅: الفجوة تحل المشكلة *فقط بالنسبة لبيانات التدريب*؛ بيانات الاختبار قد تملأ تلك الفجوة، فتعود المشكلة كما هي.

### 7. التقسيم المحلي مقابل التقسيم الشامل (Local vs Global Discretisation)

#### النص الأصلي يقول:
> "Some data mining algorithms... can be adapted so that each continuous attribute is converted to a categorical attribute at each stage of the process (e.g. at each node of the decision tree). This is known as local discretisation. An alternative approach is to use a global discretisation algorithm..."

#### الشرح المبسّط:

| المصطلح | التعريف | مثال/ملاحظة |
| --- | --- | --- |
| `Local Discretisation` | تحويل المتغير المستمر إلى فئوي **من جديد في كل عقدة (node)** من شجرة القرار | نقطة الفصل لـ `Age` قد تكون 27.3 عند عقدة، و14.1 عند عقدة أخرى تحتها |
| `Global Discretisation` | تحويل المتغير المستمر إلى فئوي **مرة واحدة لكامل مجموعة البيانات**، بشكل مستقل عن أي خوارزمية تعدين بيانات لاحقة | `Age` → `Age2` بأربع فئات A، B، C، D بنقاط فصل 16، 30، 60 محددة سلفًا |

**لماذا يهم الفرق؟** لأن `Local` قد يعطي دقة أعلى وقواعد أقل، لكنه مكلف حسابيًا لأنه يُعاد حسابه في كل عقدة. أما `Global` فيُحسب مرة واحدة فقط، ويمكن استخدام نتيجته مع أي خوارزمية أخرى غير `TDIDT`، لكن دقته وعدد قواعده قد يكونان أقل كفاءة قليلاً (كما سنرى في نتائج التجربة في القسم 7.5).

#### 💡 التشبيه:
> `Local Discretisation` أشبه بخيّاط يُعيد أخذ المقاس في كل مرحلة من تفصيل الثوب حسب الجزء الذي يعمل عليه. `Global Discretisation` أشبه بأخذ مقاس واحد شامل في البداية ثم تفصيل كل القطع بناءً عليه.
> **وجه الشبه:** إعادة القياس الموضعي المتكرر = `Local`؛ القياس المسبق الموحّد = `Global`.

### 8. إضافة التقسيم المحلي إلى TDIDT: فكرة الـ Pseudo-attribute

#### النص الأصلي يقول:
> "If a continuous attribute A has n distinct values v1, v2,..., vn... there are n − 1 possible corresponding pseudo-attributes (all binary), i.e. A<v2, A<v3, ..., A<vn."

#### الشرح المبسّط:
لكل متغير مستمر `A` له `n` قيمة مختلفة مرتبة تصاعديًا، نبني `n - 1` "متغير وهمي" (`pseudo-attribute`) ثنائي القيمة (`true`/`false`)، كل واحد منها هو اختبار من الشكل `A < vi`. نتجاهل `A < v1` لأنه لا توجد قيمة أصغر من أصغر قيمة، فسيكون دائمًا `false` ولا فائدة منه.

كل هذه المتغيرات الوهمية (بالإضافة للمتغيرات الفئوية العادية) تتنافس فيما بينها عبر معيار `Information Gain` (أو أي مقياس آخر) لاختيار الأفضل عند كل عقدة. إذا فاز `Age < 27.3` عند عقدة معينة، فهذا يعني أننا "اكتشفنا محليًا" أن نقطة الفصل المثلى لـ `Age` عند هذه العقدة هي 27.3.

**لماذا لا يُحذف المتغير المستمر بعد استخدامه؟** لأن التقسيم محلي فقط: قد نحتاج اختبارًا آخر لنفس المتغير (`Age < 14.1` مثلاً) في مستوى أعمق من الشجرة تحت الفرع "نعم" النابع من `Age < 27.3`.

```algorithm
1 | لكل متغير مستمر A عند العقدة | ترتيب تصاعدي | فرز القيم n قيمة تصاعديًا (Quicksort عادة)
2 | حساب Information Gain لكل pseudo-attribute | جدول تكرار Frequency Table | حساب المقياس لكل A<v2 ... A<vn
3 | اختيار أفضل نقطة فصل للمتغير A | مقارنة القيم | إيجاد vi التي تعطي أكبر Information Gain
4 | حساب Information Gain للمتغيرات الفئوية العادية | جدول تكرار | نفس أسلوب الفصل 5
5 | اختيار الفائز النهائي | مقارنة شاملة | اختيار المتغير أو pseudo-attribute صاحب أعلى قيمة للمقياس
```

#### نقاط التنفيذ:
- يجب حساب `Information Gain` لكل `pseudo-attribute` بشكل منفصل قبل المقارنة النهائية.
- المتغير المستمر الأصلي **يبقى متاحًا** لعقد لاحقة أعمق في نفس الفرع.

### 9. حساب الـ Information Gain لمجموعة الـ Pseudo-attributes (مثال golf)

#### النص الأصلي يقول:
> "At any node of the evolving decision tree the entropy values... of all the pseudo-attributes derived from a given continuous attribute can be calculated with a single pass through the training data."

#### الشرح المبسّط:
العملية تتم على 3 مراحل (Stages):

**المرحلة 1:** عدّ عدد الأمثلة لكل تصنيف (`class`) في الجزء الحالي من بيانات التدريب — في مثال `golf`: 9 `play` و5 `don't play` (المجموع 14). هذا العدّ **لا يعتمد** على المتغير الذي سنعالجه لاحقًا، لذا يُحسب مرة واحدة فقط لكل عقدة.

**المرحلة 2:** نمرّ على كل متغير مستمر (هنا: `temperature` و`humidity`) ونبحث عن أفضل نقطة فصل له (`cut point`) التي تُعطي أكبر قيمة للمقياس (`maxmeasure`).

**المرحلة 3:** بعد إيجاد أفضل نقطة فصل لكل متغير مستمر، نقارن هذه القيم الكبرى مع قيم المتغيرات الفئوية لاختيار الفائز الكلي عند هذه العقدة.

#### 📊 المخطط: تدفّق حساب Information Gain للمتغيرات المستمرة

#### ما هذا المخطط؟
> يوضّح تسلسل المراحل الثلاث لمعالجة كل عقدة في `TDIDT` عند وجود متغيرات مستمرة.

#### وصف العُقد:

### 10. جدول الأمثلة المرتبة (Sorted Instances Table) وجدول التكرار (Frequency Table)

#### النص الأصلي يقول:
> "The first step is to sort the values of the attribute into ascending numerical order and create a table containing just two columns... We will call this the sorted instances table."

#### الشرح المبسّط:
لبناء جداول التكرار (`Frequency Table`) لكل `pseudo-attribute` بكفاءة، نرتّب أولًا قيم المتغير المستمر تصاعديًا مع تصنيف كل مثال بجانبها (جدول `temperature` أعلاه فيه 14 صفًا). ثم نمرّ على الجدول من أعلى لأسفل، سطرًا سطرًا، ونراكم عدد الأمثلة لكل تصنيف. كل مرة نجد أن القيمة التالية أكبر من الحالية (وليست مساوية)، نعتبر تلك القيمة التالية **نقطة فصل مرشّحة** (`candidate cut point`) ونبني جدول تكرار (`2×2`) لها.

**الفائدة الأساسية:** كل جدول تكرار جديد يُشتق من الجدول السابق بتحديث بسيط جدًا (نقل مثال واحد فقط من عمود "أكبر أو يساوي" إلى عمود "أصغر من") — بدل إعادة بناء الجدول من الصفر. هذا يجعل حساب كل الـ 11 جدولاً (لمتغير `temperature` في المثال) يتم بمرور واحد فقط على البيانات المرتبة.

#### 📐 المعادلة: جدول التكرار لـ pseudo-attribute
$$
\text{Frequency Table for } (A < v_i) =
\begin{array}{c|cc|c}

```algorithm
1 | التهيئة | متغيرات | ضبط عدّاد كل تصنيف = صفر، وضبط maxmeasure لأصغر قيمة ممكنة نظريًا
2 | حلقة على كل سطر i من 1 إلى n-1 | تكرار | زيادة عدّاد class(i) بمقدار 1
```

### 12. الكفاءة الحاسوبية (Computational Efficiency)

#### النص الأصلي يقول:
> "(a) Sorting continuous values into ascending numerical order... the number of operations required is approximately a constant multiple of n × log2 n... (b) Calculating the measure value for each frequency table... (c) Number of candidate cut points..."

#### الشرح المبسّط:
هناك ثلاث نقاط عنق زجاجة (`bottlenecks`) محتملة:

**(أ) الفرز (Sorting):** هو العبء الأساسي على الخوارزمية، ويجب تنفيذه لكل متغير مستمر عند كل عقدة. خوارزمية الفرز الموصى بها هي `Quicksort` لأن عدد عملياتها يتناسب مع `n × log2 n` وليس `n²`. الفرق هائل: لمليون مثال، `n log2 n ≈ 19.9 مليون` عملية (≈ 20 ثانية بافتراض كل عملية = مايكروثانية)، بينما `n² = 10¹²` عملية (≈ 11.5 يوم!).

**(ب) حساب المقياس لكل جدول تكرار:** كل جدول صغير جدًا (4 خلايا رئيسية + مجموعان هامشيان)، لذا معالجة عدد كبير من الجداول الصغيرة تبقى قابلة للإدارة.

**(ج) عدد نقاط الفصل المرشّحة:** بشكل افتراضي يساوي عدد القيم المميزة (باستثناء الأولى)، وقد يكون كبيرًا جدًا لمجموعة بيانات ضخمة. الحل: تقليصه باستخدام معلومة التصنيف (`class information`) — انظر البند التالي.

#### 📐 المعادلة: التعقيد الزمني للفرز
$$
T_{\text{Quicksort}}(n) \approx c \cdot n \log_2 n \quad \text{مقابل} \quad T_{\text{naive}}(n) \approx n^2
$$

**الشرح:**
> `n` هو عدد الأمثلة، `c` ثابت. الفرق بين `n log2 n` و`n²` يتّسع بشكل هائل مع نمو `n` — عند `n=1{,}000{,}000` الفرق يقارب 50,000 ضعف تقريبًا.

| n | n × log₂n | n² |
| --- | --- | --- |
| 100 | 664 | 10,000 |
| 1,000 | 9,966 | 1,000,000 |
| 1,000,000 | 19,931,569 | 1,000,000,000,000 |

### 14. خوارزمية ChiMerge — الفكرة العامة

#### النص الأصلي يقول:
> "ChiMerge is a well-known algorithm for global discretisation introduced by Randy Kerber... It uses a statistical technique for discretising each continuous attribute separately."

#### الشرح المبسّط:
`ChiMerge` طريقة **تقسيم شامل (Global Discretisation)** تعتمد على اختبار إحصائي (`χ² test`) لتحديد ما إذا كان مجالان متجاوران (`intervals`) "مختلفان إحصائيًا من حيث توزيع التصنيف" أم "متشابهان بما يكفي ليُدمَجا في مجال واحد".

**الفرضية (Hypothesis) التي يختبرها ChiMerge لكل زوج مجالات متجاورين:**
> "التصنيف مستقل عن أي المجالين المتجاورين ينتمي إليه المثال."

- إذا **تأكدت** الفرضية (أي لا يوجد فرق ذو دلالة إحصائية) → **ندمج** المجالين (لا فائدة من التفريق بينهما).
- إذا **رُفضت** الفرضية (يوجد فرق ذو دلالة إحصائية) → **نُبقيهما منفصلين**.

#### 💡 التشبيه:
> تخيل جدولين متجاورين من الطلاب (مثلاً من أطول 160-170 سم، والآخر 170-180 سم)، وتسأل: "هل نسبة النجاح تختلف فعلاً بين هاتين المجموعتين، أم أن الفرق مجرد صدفة عشوائية؟" إذا كان الفرق ضئيلًا وغير دالّ إحصائيًا، تدمج المجموعتين في فئة واحدة "متوسط الطول".
> **وجه الشبه:** اختبار الفرق بين مجموعتين متجاورتين = `χ²` بين مجالين متجاورين في `ChiMerge`.

### 17. حساب القيم المتوقعة (Expected) وχ²

#### النص الأصلي يقول:
> "There are a total of colsumj occurrences of class j in the two intervals combined... we would expect rowsumi × colsumj/sum occurrences of class j in row i."

#### الشرح المبسّط:
لو كانت الفرضية (الاستقلالية) صحيحة، فإن القيمة **المتوقعة** `E` لأي خلية (صف `i`، عمود `j`) تُحسب هكذا: نسبة العمود `j` من إجمالي المجالين مضروبة في عدد أمثلة الصف `i`.

#### 📐 المعادلة: القيمة المتوقعة E
$$
E_{ij} = \frac{\text{rowsum}_i \times \text{colsum}_j}{\text{sum}}
$$

**الشرح:**
> `rowsum_i` = مجموع أمثلة المجال (الصف) `i`، `colsum_j` = مجموع أمثلة التصنيف (العمود) `j` في المجالين معًا، `sum` = المجموع الكلي للمجالين. مثال: لصف `8.7` وتصنيف `c1`: `E = 13 × 7 / 19 = 4.79`.

#### 📐 المعادلة: إحصائية χ²
$$
\chi^2 = \sum_{i,j} \frac{(O_{ij} - E_{ij})^2}{E_{ij}}
$$

**الشرح:**
> `O` = القيمة المُشاهَدة فعليًا، `E` = القيمة المتوقعة إحصائيًا تحت فرضية الاستقلال. كل ما اقتربت `O` من `E` كلما اقترب `χ²` من الصفر (دعم للفرضية = دمج المجالين). كلما ابتعدت `O` عن `E` كبُرت قيمة `χ²` (رفض للفرضية = إبقاء المجالين منفصلين).

**⚠️ تفصيل تقني مهم:** إذا كانت `E < 0.5` لأي خلية، يُستبدل المقام بـ `0.5` بدل `E` نفسها (تصحيح إحصائي قياسي لتفادي القسمة على قيم صغيرة جدًا تُضخّم `χ²` بشكل مصطنع).

#### مثال محسوب (المجالان 8.7 و12.1):

| Value of A | O(c1) | E(c1) | Val | O(c2) | E(c2) | Val | O(c3) | E(c3) | Val |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 8.7 | 6 | 4.79 | 0.31 | 0 | 0.74 | 0.74 | 1 | 1.47 | 0.15 |
| 12.1 | 7 | 8.21 | 0.18 | 2 | 1.26 | 0.43 | 3 | 2.53 | 0.09 |

مجموع كل قيم `Val` = **χ² = 1.89**.

### 18. عتبة القرار (Threshold) ومقارنتها مع χ²

#### النص الأصلي يقول:
> "This value is then compared with a threshold value T... For the current example we will use a significance level of 90%... this gives a threshold value of 4.61."

#### الشرح المبسّط:
نقارن قيمة `χ²` المحسوبة مع **عتبة (Threshold)** تعتمد على أمرين: **مستوى الدلالة الإحصائية** (`significance level`، عادة 90%، 95%، أو 99%) و**درجات الحرية** (`degrees of freedom = عدد التصنيفات − 1`).

- إذا `χ² < Threshold` → دعم لفرضية الاستقلال → **دمج المجالين**.
- إذا `χ² > Threshold` → رفض للفرضية → **إبقاؤهما منفصلين**.

**مثال:** بدرجتي حرية (3 تصنيفات − 1 = 2) ومستوى دلالة 90%، العتبة = **4.61**. بما أن `χ² = 1.89 < 4.61` في المثال أعلاه، فإن المجالين `8.7` و`12.1` **يُدمَجان** لو كانا هما الزوج الأصغر قيمة (لكن في المثال الفعلي كان الزوج الأصغر هو `1.4` و`1.8` بقيمة `χ² = 1.08`).

#### ⚖️ المقايضة: اختيار مستوى الدلالة

| | مستوى دلالة منخفض (مثلاً 90%) | مستوى دلالة مرتفع (مثلاً 99%) |
| --- | --- | --- |
| المزايا | عتبة أصغر، دمج أسهل، عدد فئات نهائي أقل | عتبة أكبر، معيار أكثر تشددًا لإبقاء الفروق الحقيقية فقط |
| العيوب | قد يُدمج فروق حقيقية بالخطأ | قد ينتج عدد فئات كبير جدًا (Overfitting محتمل) |
| متى تختاره | عندما تريد عدد فئات نهائي صغير وقابل للإدارة | عندما تريد الحفاظ على أدق الفروق الإحصائية الممكنة |

### 19. عملية الدمج التكرارية الكاملة

#### النص الأصلي يقول:
> "ChiMerge selects the smallest value of χ2... The value 1.08 is less than the threshold value so the independence hypothesis is supported and the two intervals are merged."

#### الشرح المبسّط:
الخوارزمية **لا** تُدمج أي زوج عشوائي؛ بل في كل خطوة تبحث عن **الزوج المتجاور صاحب أصغر قيمة χ²** في كامل الجدول (لأنه الأكثر "تشابهًا إحصائيًا")، وتدمجه إن كانت قيمته أقل من العتبة. يتكرر هذا حتى لا يبقى أي زوج قيمته أقل من العتبة — عندها تتوقف الخوارزمية وتكون النتيجة النهائية هي الفئات المستقرة إحصائيًا.

**تفصيل مهم:** بعد كل دمج، لا يلزم إعادة حساب **كل** قيم `χ²` من جديد — فقط الأزواج التي أصبح المجال المدموج طرفًا فيها (المجال الجديد له علاقتان فقط: مع الذي يسبقه ومع الذي يليه).

#### 🔍 تتبع التنفيذ: أول خطوتي دمج في مثال ChiMerge

**المدخل:** الجدول الأولي بـ 11 مجالاً (Figure 7.12(a))، أصغر χ² = 1.08 بين `1.4` و`1.8`.

| الخطوة | العملية | الحالة |
| --- | --- | --- |
| 1 | مقارنة كل قيم χ² العشر | أصغر قيمة = 1.08 (بين 1.4 و1.8)، والعتبة = 4.61 |
| 2 | 1.08 < 4.61 → دمج | المجالان 1.4 و1.8 يُدمجان تحت التسمية `1.4` (الأصغر) |
| 3 | إعادة حساب χ² للأزواج المتأثرة فقط | جدول جديد من 10 مجالات (Figure 7.12(b), (c)) |
| 4 | أصغر χ² جديد = 1.20 (بين 87.1 و89.0) | 1.20 < 4.61 → دمج المجالين 87.1 و89.0 |

**النتيجة:** بعد استمرار هذه العملية، تصل الخوارزمية إلى جدول نهائي من 3 مجالات فقط (Figure 7.12(d)) لا يوجد بينها أي زوج χ² أقل من 4.61.

#### الشرح المبسّط:
هذه هي النتيجة النهائية: متغير `A` المستمر أصبح متغيرًا فئويًا بثلاث قيم فقط، جاهزًا للاستخدام في أي خوارزمية تعدين بيانات (وليس `TDIDT` فقط).

#### مهم للامتحان ⚠️:
> مشكلة عملية مهمة: قيمة جديدة في بيانات الاختبار قد تكون **أصغر من أصغر قيمة رأيناها في التدريب** (أقل من 1.3) أو **أكبر من أكبر قيمة** (أكبر من 89.0 رغم أن الفئة الأخيرة معرّفة بصيغة `A ≥ 87.1`). يجب اتخاذ قرار: إما إلحاقها بأقرب فئة طرفية، أو اعتبارها "غير قابلة للتصنيف" (`unclassifiable`).

### 20. درجات الحرية (Degrees of Freedom) وجدول العتبات

#### النص الأصلي يقول:
> "In the general case of a contingency table with N rows and M columns the number of independent values in the main body of the table is (N −1)×(M−1)... for ChiMerge... the number of degrees of freedom is (2 − 1) × (number of classes − 1) = number of classes − 1."

#### الشرح المبسّط:
في `ChiMerge`، عدد الصفوف دائمًا 2 (مجالان متجاوران فقط في كل مقارنة)، وعدد الأعمدة = عدد التصنيفات. لذلك درجات الحرية دائمًا = `عدد التصنيفات − 1`. لماذا؟ لأنه بمجرد معرفة المجاميع الهامشية (مجموع كل صف ومجموع كل عمود)، فإن كل الخلايا محدَّدة تلقائيًا **إلا** عدد قليل منها يمكن ملؤه "بحرية" — والباقي يُحسب رياضيًا من المجاميع.

كلما زادت درجات الحرية، ارتفعت قيمة العتبة (انظر الجدول الكامل في المحاضرة، Figure 7.13).

#### 🤔 تفعيل الفهم (اسأل نفسك):
> **سؤال:** إذا كان لدينا 5 تصنيفات بدل 3، هل ستكون عملية الدمج أسهل أم أصعب (بنفس مستوى الدلالة)؟
> **لماذا هذا مهم؟** درجات الحرية = 5-1 = 4، والعتبة عند 90% ترتفع من 4.61 إلى 7.78 (انظر جدول Figure 7.13) — أي يصبح الدمج **أسهل نسبيًا** لأن العتبة أعلى وأكثر قيم χ² ستكون أقل منها.

### 21. minIntervals وmaxIntervals

#### النص الأصلي يقول:
> "Kerber [7] proposed setting two values, minIntervals and maxIntervals. The form of the algorithm always merges the pair of intervals with the lowest value of χ2 as long as the number of intervals is more than maxIntervals..."

#### الشرح المبسّط:
مشكلة عملية: النتيجة النهائية قد تكون عدد فئات كبير جدًا (آلاف) أو قليل جدًا (فئة واحدة فقط — ما يعني أن المتغير عمليًا مستقل عن التصنيف وربما يجب حذفه). الحل: تحديد حدّين:
- `maxIntervals`: طالما عدد الفئات أكبر منه، **استمر بالدمج إجباريًا** (حتى لو كانت قيمة χ² أكبر من العتبة!).
- `minIntervals`: توقف عن الدمج إذا وصل عدد الفئات لهذا الحدّ، حتى لو كانت أصغر قيمة χ² لا تزال أقل من العتبة.

**قيم مقترحة عمليًا:** `minIntervals` = 2 أو 3، و`maxIntervals` = 20.

#### الفهم الخاطئ الشائع ❌: القاعدة الإحصائية النظرية (مقارنة χ² بالعتبة) هي التي تحكم دائمًا عدد الفئات النهائي.
#### الفهم الصحيح ✅: حدود `minIntervals`/`maxIntervals` قد **تتجاوز** القرار الإحصائي البحت لأسباب عملية بحتة (تفادي عدد فئات غير عملي) — وهذا "صعب التبرير نظريًا" كما يذكر النص صراحة، لكنه مفيد عمليًا.

### 22. مقارنة تجريبية: Local vs Global Discretisation مع TDIDT

#### النص الأصلي يقول:
> "The percentage of correct classifications for the global discretisation approach is comparable with those achieved by local discretisation. However, local discretisation seems to produce an appreciably smaller number of rules..."

#### الشرح المبسّط:
تجربة على 7 مجموعات بيانات من UCI Repository باستخدام `10-fold cross-validation` قارنت بين:
- التقسيم المحلي (المدمج داخل `TDIDT` مباشرة).
- التقسيم الشامل بـ `ChiMerge` (90% مستوى دلالة) ثم تطبيق `TDIDT` على البيانات الفئوية الناتجة.

**النتيجة الرئيسية:**
- **الدقة (correct %):** متقاربة جدًا بين الطريقتين.
- **عدد القواعد:** `Local Discretisation` ينتج عددًا أقل بشكل ملحوظ من القواعد، خصوصًا في `pima-indians` و`sick-euthyroid`.

| Dataset | Local: عدد القواعد | Local: دقة% | Global: عدد القواعد | Global: دقة% |
| --- | --- | --- | --- | --- |
| glass | 38.3 | 69.6 | 88.2 | 72.0 |
| pima-indians | 121.9 | 69.8 | 328.0 | 74.0 |
| sick-euthyroid | 72.7 | 96.6 | 265.1 | 96.6 |

**لماذا يهم هذا؟** لأن عدد القواعد الأقل يعني شجرة/نموذج أبسط وأسهل للتفسير البشري، حتى لو كانت الدقة متشابهة. لكن `Global` له ميزة عملية أخرى: **يُحسَب مرة واحدة فقط** ويمكن استخدامه لأي خوارزمية تصنيف أخرى غير `TDIDT`.

#### ⚖️ المقايضة الشاملة: Local مقابل Global Discretisation

| | Local Discretisation | Global Discretisation (ChiMerge) |
| --- | --- | --- |
| المزايا | عدد قواعد أقل، دقة مماثلة أو أفضل قليلاً | يُحسَب مرة واحدة، قابل لإعادة الاستخدام مع أي خوارزمية |
| العيوب | مكلف حسابيًا (يُعاد الحساب في كل عقدة) | عدد قواعد أكبر عادة لنفس مستوى الدقة تقريبًا |
| متى تختاره | عند استخدام `TDIDT` فقط وتريد تفسيرًا أبسط | عند وجود عدة خوارزميات تعدين بيانات تحتاج لنفس البيانات المُقسَّمة |

## الجزء الثاني: ملخص منظم

### أهم التعاريف والمفاهيم

| المصطلح | التعريف | مثال/ملاحظة |
| --- | --- | --- |
| `Discretisation` | تحويل متغير مستمر إلى متغير فئوي بعدد محدود من الفئات | `length` → 3 مجالات |
| `Cut Point` / `Cut Value` | القيمة الفاصلة بين مجالين متتاليين | 7، 12، 20 في المثال الافتتاحي |
| `Equal Width Intervals` | تقسيم مدى القيم إلى أجزاء متساوية العرض | 3 أجزاء بعرض ≈2.1 لكل جزء |
| `Equal Frequency Intervals` | تقسيم القيم بحيث يحتوي كل مجال على نفس عدد الأمثلة | نفس فكرة لكن بعدد أمثلة متساوٍ |
| `Local Discretisation` | تقسيم يُعاد حسابه عند كل عقدة في الشجرة | `TDIDT` مع pseudo-attributes |
| `Global Discretisation` | تقسيم يُحسب مرة واحدة لكامل البيانات | `ChiMerge` |
| `Pseudo-attribute` | متغير ثنائي وهمي من الشكل `A < v` يُبنى من متغير مستمر | `temperature < 65` |
| `Sorted Instances Table` | جدول القيم مرتبة تصاعديًا مع تصنيفها | جدول temperature/class |
| `Frequency Table` | جدول تكرار 2×2 (أو أكثر) يُستخدم لحساب المقاييس | Figure 7.3(a,b,c) |
| `ChiMerge` | خوارزمية تقسيم شامل تعتمد اختبار χ² لدمج المجالات المتشابهة إحصائيًا | خوارزمية Kerber |
| `Contingency Table` | جدول توافقي يقارن التكرار المُشاهَد بين مجالين وتصنيفاتهم | Figure 7.10(a) |
| `Observed (O)` | التكرار الفعلي المُشاهَد في البيانات | من الجدول التوافقي مباشرة |
| `Expected (E)` | التكرار المتوقع رياضيًا تحت فرضية الاستقلال | `rowsum × colsum / sum` |
| `χ² (Chi-square)` | إحصائية تقيس الفرق بين O وE لتحديد الاستقلال الإحصائي | `Σ(O-E)²/E` |
| `Degrees of Freedom` | عدد القيم القابلة للتحديد بحرية داخل الجدول | `عدد التصنيفات − 1` في ChiMerge |
| `Threshold (T)` | القيمة الحدّية لمقارنة χ² المحسوبة | 4.61 عند درجتي حرية و90% |
| `minIntervals` / `maxIntervals` | حدود عملية لعدد الفئات النهائي | مقترح: 2-3 و20 |

### نقاط ذهبية

1. أي طريقة تقسيم (`discretisation`) تعاني من "الحساسية عند نقاط الفصل" — لا يوجد حل مثالي 100%.
2. الطرق التي تستخدم معلومة التصنيف (`ChiMerge`، `TDIDT` المحلي) أفضل عمومًا من الطرق التي تتجاهله (`Equal Width`/`Equal Frequency`).
3. `Local Discretisation` غالبًا يعطي عدد قواعد أقل، بينما `Global` أكثر قابلية لإعادة الاستخدام.
4. الكفاءة الحاسوبية (خصوصًا الفرز) هي القيد الأساسي على حجم البيانات القابل للمعالجة.

### أخطاء شائعة

- الاعتقاد أن زيادة عدد الفئات دائمًا تحسّن الدقة (في الواقع قد تسبب Overfitting وقواعد غير قابلة للتعميم).
- نسيان تصحيح `E < 0.5 → استخدم 0.5` عند حساب `χ²`.
- الخلط بين `minIntervals`/`maxIntervals` وبين العتبة الإحصائية — الأولى قيود عملية صارمة، والثانية معيار إحصائي.

### خطوات وإجراءات المحاضرة (Algorithm Blocks مستقلة)

#### ⚙️ الخطوات / الخوارزمية: التقسيم المحلي في TDIDT عند كل عقدة

#### ما هدف هذه العملية؟
> إيجاد أفضل pseudo-attribute (أو متغير فئوي عادي) للتقسيم عند عقدة معينة في شجرة القرار.

```algorithm
1 | لكل متغير مستمر A | فرز | ترتيب الأمثلة تصاعديًا حسب A
2 | حساب المقياس لكل قيمة مميزة | Frequency Table | بناء n-1 pseudo-attribute وحساب Information Gain لكل واحد
3 | اختيار أفضل نقطة فصل لكل A | مقارنة | إيجاد vi صاحبة أعلى Information Gain، وتسجيلها كـ maxmeasure وcutvalue
4 | حساب المقياس للمتغيرات الفئوية | Frequency Table عادي | بنفس طريقة الفصل 5 من قبل
5 | الاختيار النهائي | مقارنة شاملة | اختيار أعلى قيمة من بين كل المتغيرات وpseudo-attributes
```

#### نقاط التنفيذ:
- الخطوات 1-3 تتكرر لكل متغير مستمر على حدة عند نفس العقدة.
- المتغير المستمر لا يُستبعد بعد الاستخدام؛ يبقى متاحًا للعقد الأعمق.

---

#### ⚙️ الخطوات / الخوارزمية: خوارزمية معالجة الجدول المرتّب (Figure 7.2)

#### ما هدف هذه العملية؟
> حساب maxmeasure وcutvalue لمتغير مستمر واحد بمرور واحد فقط على البيانات.

```algorithm
1 | التهيئة | متغيرات العداد | عدّاد كل تصنيف = 0، maxmeasure = أصغر قيمة ممكنة
2 | لكل i من 1 إلى n-1 | تكرار | زيادة عدّاد class(i)
3 | فحص value(i) < value(i+1) | شرط | إذا تحقق → المتابعة، وإلا → السطر التالي
4 | بناء جدول تكرار لـ Var<value(i+1) | Frequency Table | حساب O وE لكل خلية
5 | حساب value المقياس | Information Gain مثلاً | من جدول التكرار
6 | تحديث maxmeasure وcutvalue إذا لزم | مقارنة | إذا measure > maxmeasure → تحديث القيمتين
```

#### نقاط التنفيذ:
- التوقف عند `n-1` وليس `n` لعدم وجود صف تالٍ للمقارنة مع الصف الأخير.

---

#### ⚙️ الخطوات / الخوارزمية: ChiMerge الكاملة (Figure 7.14)

#### ما هدف هذه العملية؟
> تقسيم متغير مستمر بشكل شامل (Global) إلى فئات مستقرة إحصائيًا اعتمادًا على اختبار χ².

```algorithm
1 | ضبط الإعدادات الأولية | minIntervals/maxIntervals | تحديد 2 ≤ minIntervals ≤ maxIntervals
2 | تحديد العتبة | جدول χ² الإحصائي | اختيار مستوى دلالة (مثلاً 90%) وإيجاد العتبة حسب درجات الحرية
3 | فرز قيم المتغير | Quicksort | ترتيب تصاعدي
4 | بناء جدول التكرار الأولي | Frequency Table | صف لكل قيمة مميزة، عمود لكل تصنيف
5 | فحص شرط التوقف الأول | مقارنة عدد الصفوف | إذا عدد الصفوف = minIntervals → توقف
6 | حساب χ² لكل زوج متجاور | O وE لكل خلية | جمع (O-E)²/E لكل زوج
7 | إيجاد أصغر χ² | مقارنة شاملة | تحديد الزوج الأكثر تشابهًا إحصائيًا
8 | فحص شرط الدمج | مقارنة بالعتبة أو maxIntervals | إذا (χ² < العتبة) أو (عدد الصفوف > maxIntervals) → دمج وإعادة الخطوة 5، وإلا توقف
```

#### نقاط التنفيذ:
- تصحيح `E < 0.5 → استخدم 0.5` يجب تطبيقه في كل حساب χ² بلا استثناء.
- شرط `maxIntervals` قد يجبر الدمج حتى لو تجاوزت χ² العتبة الإحصائية.

2. الزوج ذو أعلى χ² هو الأكثر اختلافًا، ولذلك **يجب إبقاؤه منفصلاً** لا دمجه.
3. هذا خلط شائع بين معنى "التشابه" و"الاختلاف" الإحصائي.

### سؤال تصحيح 5 (`wrong_formula`)
**الكود/الحساب (يحتوي خطأ):**
```r
chi_sq_val <- (O - E)^2 / O   # (1) division by O instead of E
```
**اكتشف الخطأ:** المقام في صيغة `χ²` يجب أن يكون `E` (القيمة المتوقعة)، وليس `O` (القيمة المُشاهَدة).
**التصحيح:**
```r

```
**شرح الحل:**
1. الصيغة الرسمية `Σ(O-E)²/E` تُطبَّع الفرق بحجم القيمة **المتوقعة**، وليس المُشاهَدة.
2. القسمة على `O` تُعطي وزنًا غير متسق للخلايا ذات قيم `O` صغيرة أو صفرية (خطر القسمة على صفر مباشرة).
3. تذكّر أيضًا: إذا كانت `E < 0.5`، استبدل المقام بـ `0.5` بدل `E` نفسها.

**الكود/الحساب (يحتوي خطأ):**

### السؤال 1: ما هو `Discretisation` ولماذا نحتاجه؟
**نموذج الإجابة:**
1. **التعريف:** تحويل متغير مستمر إلى متغير فئوي بعدد محدود من الفئات/المجالات.
2. **المكونات/الشروط:** يتطلب اختيار عدد المجالات ونقاط الفصل (cut points) بينها.
3. **مثال:** `temperature` → `low`, `medium`, `high`.
4. **متى نستخدم:** عند استخدام خوارزمية تتطلب متغيرات فئوية حصرًا مثل `TDIDT`.

---

### السؤال 2: قارن بين `Local` و`Global Discretisation`.
**نموذج الإجابة:**
1. **التعريف:** `Local` يُعاد حسابه عند كل عقدة في شجرة القرار؛ `Global` يُحسب مرة واحدة لكامل البيانات.
2. **المكونات/الشروط:** `Local` مرتبط بخوارزمية معينة (مثل `TDIDT`)؛ `Global` مستقل عن أي خوارزمية لاحقة.
3. **مثال:** `TDIDT` مع pseudo-attributes (Local)، `ChiMerge` (Global).
4. **متى نستخدم:** `Local` عند أولوية تقليل عدد القواعد؛ `Global` عند الحاجة لإعادة الاستخدام مع خوارزميات متعددة.

---

### السؤال 3: اشرح فكرة الـ `pseudo-attribute`.
**نموذج الإجابة:**
1. **التعريف:** متغير ثنائي (`true`/`false`) من الشكل `A < v` مُشتق من متغير مستمر.
2. **المكونات/الشروط:** لكل قيمة مميزة (باستثناء الأصغر) يُبنى pseudo-attribute واحد.
3. **مثال:** `Age < 27.3`.
4. **متى نستخدم:** عند تطبيق `TDIDT` مع `Local Discretisation` عند كل عقدة.

---

### السؤال 4: ما الفرضية التي يختبرها `ChiMerge`؟
**نموذج الإجابة:**
1. **التعريف:** أن التصنيف مستقل عن أي المجالين المتجاورين ينتمي إليه المثال.
2. **المكونات/الشروط:** تُختبر عبر إحصائية χ² مقارنة بعتبة معتمدة على مستوى الدلالة ودرجات الحرية.
3. **مثال:** مجالان بقيمة χ²=1.08 وعتبة 4.61 → دعم للفرضية → دمج.
4. **متى نستخدم:** عند بناء Global Discretisation باستخدام ChiMerge.

---

### السؤال 5: كيف تُحسب القيمة المتوقعة `E` في جدول توافقي؟
**نموذج الإجابة:**
1. **التعريف:** `E = (rowsum × colsum) / total sum`.
2. **المكونات/الشروط:** `rowsum` مجموع المجال، `colsum` مجموع التصنيف، `total` مجموع المجالين المقارنَين.
3. **مثال:** `E(8.7,c1) = 13×7/19 = 4.79`.
4. **متى نستخدم:** في كل خلية من الجدول التوافقي قبل حساب χ².

---

### السؤال 6: ما دور `minIntervals` و`maxIntervals`؟
**نموذج الإجابة:**
1. **التعريف:** حدّان عمليان يحكمان أدنى وأعلى عدد فئات نهائي مسموح به.
2. **المكونات/الشروط:** `maxIntervals` يُجبر الدمج حتى لو تجاوزت χ² العتبة؛ `minIntervals` يمنع الدمج إذا وصل العدد له.
3. **مثال:** minIntervals=2، maxIntervals=20.
4. **متى نستخدم:** عندما تُنتج القاعدة الإحصائية البحتة عدد فئات غير عملي (كبير جدًا أو صغير جدًا).

---

### السؤال 7: لماذا يُفضَّل `Quicksort` في هذا السياق؟
**نموذج الإجابة:**
1. **التعريف:** خوارزمية فرز بتعقيد زمني `O(n log n)` في المتوسط.
2. **المكونات/الشروط:** تُطبَّق مرة واحدة لكل متغير مستمر عند كل عقدة.
3. **مثال:** لـ1,000,000 مثال: 19.9 مليون عملية تقريبًا مقابل 10¹² لخوارزمية `n²`.
4. **متى نستخدم:** كلما كانت مجموعة البيانات كبيرة، حيث يكبر الفرق بين `n log n` و`n²` بشكل هائل.

---

### السؤال 8: اشرح الفرق بين `Equal Width` و`Equal Frequency Intervals` من حيث نقطة الضعف المشتركة.
**نموذج الإجابة:**
1. **التعريف:** كلاهما طريقتان لا تستخدمان معلومة التصنيف.
2. **المكونات/الشروط:** الأولى تقسم المدى بالتساوي، الثانية تقسم عدد الأمثلة بالتساوي.
3. **مثال:** `length` من 0.3 إلى 6.6.
4. **متى نستخدم:** كخطوة أولية بسيطة، مع إدراك أن الطرق التي تستخدم التصنيف (`ChiMerge`, `TDIDT` المحلي) عادة أفضل.

---

### السؤال 9: ما معنى "over-sensitivity" في سياق نقاط الفصل؟
**نموذج الإجابة:**
1. **التعريف:** معاملة قيمتين متقاربتين جدًا معاملة مختلفة كليًا بسبب وقوعهما على جانبي نقطة فصل.
2. **المكونات/الشروط:** تحدث عند أي اختيار لنقطة فصل، بغض النظر عن الطريقة.
3. **مثال:** `2.39999` مقابل `2.40001` عند حدّ `2.4`.
4. **متى نستخدم (أو نتجنبها):** يمكن تخفيفها (وليس حلها كليًا) باختيار نقطة فصل داخل فجوة في بيانات التدريب.

---

### السؤال 10: صف خطوات خوارزمية معالجة الجدول المرتب (Figure 7.2) بإيجاز.
**نموذج الإجابة:**
1. **التعريف:** خوارزمية لحساب maxmeasure وcutvalue لمتغير مستمر واحد بمرور واحد فقط.
2. **المكونات/الشروط:** تهيئة العدّادات، حلقة من 1 إلى n-1، شرط `value(i)<value(i+1)`، بناء جدول تكرار وحساب المقياس عند تحقق الشرط.
3. **مثال:** مثال `golf`/`temperature` بـ11 نقطة فصل مرشّحة.
4. **متى نستخدم:** عند كل متغير مستمر عند كل عقدة في `TDIDT`.

---

### السؤال 11: لماذا تختلف نتائج التجربة (Figure 7.16) بين مجموعات البيانات المختلفة؟
**نموذج الإجابة:**
1. **التعريف:** طبيعة البيانات (عدد المتغيرات المستمرة، توزيع القيم، عدد التصنيفات) تؤثر على أداء كل طريقة.
2. **المكونات/الشروط:** حجم البيانات وعدد المتغيرات المستمرة يزيدان أثر الفرق بين Local وGlobal.
3. **مثال:** `pima-indians` و`sick-euthyroid` أظهرا أكبر فرق في عدد القواعد.
4. **متى نستخدم:** لا توجد إجابة واحدة صحيحة دائمًا؛ يجب اختبار الطريقتين تجريبيًا على البيانات الفعلية.

---

### السؤال 12: ما العلاقة بين اختبار χ² المستخدم في `ChiMerge` واختبار الفرضيات الإحصائية العام؟
**نموذج الإجابة:**
1. **التعريف:** `ChiMerge` يستخدم نفس منطق اختبار الفرضيات: فرضية عدم (استقلال)، إحصائية اختبار (χ²)، ومستوى دلالة/عتبة لاتخاذ القرار.
2. **المكونات/الشروط:** رفض الفرضية عندما تتجاوز الإحصائية العتبة؛ قبولها (عدم رفضها) في الحالة المعاكسة.
3. **مثال:** استخدام جدول درجات الحرية والدلالة (Figure 7.13) هو نفسه المستخدم في اختبارات χ² العامة.
4. **متى نستخدم:** هذا الرابط يوضّح أن `ChiMerge` تطبيق عملي مباشر لمفهوم إحصائي عام تعلمناه سابقًا في اختبار الفرضيات.

```r
# Reference implementation of ideas in Chapter 7: Continuous Attributes

# --- Part 1: Best cut point for one continuous attribute (Local Discretisation idea) ---
find_best_cut_point <- function(values, classes) {
  # Sort values ascending, keep classes aligned
  ord <- order(values)                 # get sorting order
  sorted_values  <- values[ord]        # apply order to values
  sorted_classes <- classes[ord]       # apply same order to classes
  n <- length(sorted_values)           # number of instances

  class_levels <- unique(classes)      # distinct class labels
  counts <- setNames(rep(0, length(class_levels)), class_levels)  # running counts

  best_measure <- -Inf                 # start below any possible measure
  best_cut <- NA                       # no cut point found yet

  for (i in 1:(n - 1)) {                              # loop up to n-1 (Figure 7.2 rule)
    counts[sorted_classes[i]] <- counts[sorted_classes[i]] + 1  # update running count

    if (sorted_values[i] < sorted_values[i + 1]) {     # candidate cut point condition
      cut_value <- sorted_values[i + 1]                # candidate cut value
      total_counts <- table(sorted_classes)            # totals for entire node (Stage 1)

      below <- counts                                  # counts for value < cut_value
      above <- total_counts - counts                   # counts for value >= cut_value

      measure <- information_gain(below, above, total_counts)  # compute IG

      if (measure > best_measure) {                    # update best if improved
        best_measure <- measure
        best_cut <- cut_value
      }
    }
  }
  list(cutvalue = best_cut, maxmeasure = best_measure)  # return both values
}

# --- Part 2: Information Gain helper (frequency-table based) ---
information_gain <- function(below, above, total_counts) {
  n_below <- sum(below)                # total instances below cut
  n_above <- sum(above)                # total instances at/above cut
  n_total <- n_below + n_above         # grand total

  entropy <- function(counts) {        # standard entropy formula
    p <- counts / sum(counts)          # class proportions
    p <- p[p > 0]                      # avoid log(0)
    -sum(p * log2(p))                  # entropy value
  }

  parent_entropy <- entropy(sum_by_names(below, above))  # entropy before split
  weighted_child_entropy <-
    (n_below / n_total) * entropy(below) +               # weighted entropy: below
    (n_above / n_total) * entropy(above)                 # weighted entropy: above

  parent_entropy - weighted_child_entropy   # Information Gain = reduction in entropy
}

sum_by_names <- function(a, b) {         # helper: element-wise sum by class name
  a + b
}

# --- Part 3: ChiMerge (Global Discretisation) ---
chi_square_pair <- function(row1, row2) {
  total <- row1 + row2                     # combined counts per class
  sum_total <- sum(total)                  # grand total for the pair
  rowsum1 <- sum(row1)                     # row sum for interval 1
  rowsum2 <- sum(row2)                     # row sum for interval 2

  chi_sq <- 0                              # accumulator
  for (j in seq_along(total)) {            # loop over each class column
    E1 <- rowsum1 * total[j] / sum_total   # expected value, interval 1
    E2 <- rowsum2 * total[j] / sum_total   # expected value, interval 2
    E1 <- max(E1, 0.5)                     # apply E<0.5 correction
    E2 <- max(E2, 0.5)                     # apply E<0.5 correction
    chi_sq <- chi_sq +
      (row1[j] - E1)^2 / E1 +              # contribution from interval 1
      (row2[j] - E2)^2 / E2                # contribution from interval 2
  }
  chi_sq
}

chimerge <- function(freq_table, threshold, min_intervals = 2, max_intervals = 20) {
  # freq_table: matrix, rows = intervals (sorted), cols = classes
  repeat {
    n_rows <- nrow(freq_table)                       # current number of intervals
    if (n_rows <= min_intervals) break                # stop condition (c)

    chi_values <- sapply(1:(n_rows - 1), function(i)  # chi-square for every adjacent pair
      chi_square_pair(freq_table[i, ], freq_table[i + 1, ]))

    min_idx <- which.min(chi_values)                  # index of smallest chi-square
    min_chi <- chi_values[min_idx]                    # smallest chi-square value

    if (min_chi < threshold || n_rows > max_intervals) {  # merge condition (f)
      merged_row <- freq_table[min_idx, ] + freq_table[min_idx + 1, ]  # sum the two rows
      freq_table <- freq_table[-(min_idx + 1), ]      # remove the second row
      freq_table[min_idx, ] <- merged_row             # replace first row with merged sum
    } else {
      break                                           # no more merging possible
    }
  }
  freq_table
}
```

```python
# === script.py ===
# Reference implementation of ideas in Chapter 7: Continuous Attributes
import numpy as np

# --- Part 1: Best cut point for one continuous attribute ---
def find_best_cut_point(values, classes):
    order = np.argsort(values)                 # get sorting order (ascending)
    sorted_values = np.array(values)[order]    # apply order to values
    sorted_classes = np.array(classes)[order]  # apply same order to classes
    n = len(sorted_values)                     # number of instances

    class_levels = sorted(set(classes))                    # distinct class labels
    total_counts = {c: list(classes).count(c) for c in class_levels}  # Stage 1 totals
    counts = {c: 0 for c in class_levels}                  # running counts

    best_measure = float("-inf")   # start below any possible measure
    best_cut = None                # no cut point found yet

    for i in range(n - 1):                                 # loop up to n-1
        counts[sorted_classes[i]] += 1                     # update running count

        if sorted_values[i] < sorted_values[i + 1]:         # candidate cut point condition
            cut_value = sorted_values[i + 1]                # candidate cut value
            below = counts.copy()                           # counts for value < cut_value
            above = {c: total_counts[c] - below[c] for c in class_levels}  # counts >= cut

            measure = information_gain(below, above)        # compute Information Gain

            if measure > best_measure:                      # update best if improved
                best_measure = measure
                best_cut = cut_value

    return {"cutvalue": best_cut, "maxmeasure": best_measure}


# --- Part 2: Information Gain helper ---
def entropy(counts_dict):
    total = sum(counts_dict.values())          # total instances
    if total == 0:
        return 0.0
    ps = [v / total for v in counts_dict.values() if v > 0]  # class proportions (avoid 0)
    return -sum(p * np.log2(p) for p in ps)    # entropy value


def information_gain(below, above):
    n_below = sum(below.values())              # total instances below cut
    n_above = sum(above.values())              # total instances at/above cut
    n_total = n_below + n_above                # grand total

    parent = {c: below[c] + above[c] for c in below}  # combined counts (parent node)
    parent_entropy = entropy(parent)                  # entropy before split

    weighted_child_entropy = (
        (n_below / n_total) * entropy(below) +        # weighted entropy: below
        (n_above / n_total) * entropy(above)          # weighted entropy: above
    )
    return parent_entropy - weighted_child_entropy    # Information Gain


# --- Part 3: ChiMerge (Global Discretisation) ---
def chi_square_pair(row1, row2):
    total = row1 + row2                       # combined counts per class
    sum_total = total.sum()                   # grand total for the pair
    rowsum1 = row1.sum()                      # row sum for interval 1
    rowsum2 = row2.sum()                      # row sum for interval 2

    chi_sq = 0.0                              # accumulator
    for j in range(len(total)):               # loop over each class column
        E1 = max(rowsum1 * total[j] / sum_total, 0.5)  # expected value + E<0.5 fix
        E2 = max(rowsum2 * total[j] / sum_total, 0.5)  # expected value + E<0.5 fix
        chi_sq += (row1[j] - E1) ** 2 / E1             # contribution from interval 1
        chi_sq += (row2[j] - E2) ** 2 / E2             # contribution from interval 2
    return chi_sq


def chimerge(freq_table, threshold, min_intervals=2, max_intervals=20):
    # freq_table: 2D numpy array, rows = intervals (sorted), cols = classes
    freq_table = freq_table.copy()
    while True:
        n_rows = freq_table.shape[0]              # current number of intervals
        if n_rows <= min_intervals:                # stop condition (c)
            break

        chi_values = [
            chi_square_pair(freq_table[i], freq_table[i + 1])  # chi-square per adjacent pair
            for i in range(n_rows - 1)
        ]
        min_idx = int(np.argmin(chi_values))       # index of smallest chi-square
        min_chi = chi_values[min_idx]               # smallest chi-square value

        if min_chi < threshold or n_rows > max_intervals:  # merge condition (f)
            merged_row = freq_table[min_idx] + freq_table[min_idx + 1]  # sum two rows
            freq_table = np.delete(freq_table, min_idx + 1, axis=0)     # remove 2nd row
            freq_table[min_idx] = merged_row                            # replace with sum
        else:
            break                                    # no more merging possible

    return freq_table
```

> **المكتبات المطلوبة (Imports):** لا تتطلب مكتبات خارجية إضافية في R (دوال أساسية فقط). في Python: `numpy` فقط.
> **الناتج المتوقع:** `find_best_cut_point` تُعيد `cutvalue` و`maxmeasure` لأفضل نقطة فصل. `chimerge` تُعيد جدول تكرار مُدمَج نهائيًا (عدد صفوف = عدد الفئات النهائية).
